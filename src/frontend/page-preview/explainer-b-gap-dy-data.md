# 页面交互差异 - dy-data（抖音经营数据引擎）

> 生成时间: 2026-07-20 12:05 +08:00
> Skill: page-explainer
> 说明: 本文件只记录 Phase 3 已证实但不属于本轮页面说明闭环的范围差异；当前没有要求立即回环 page-designer 的 `design_gap` 或 `logic_conflict`。

## 差异清单

### GAP-P3-01: 普通管理员写权限尚未在全部后台模块统一收口

- **分类**: `out_of_scope`
- **所在页面**: 账号管理 (`/admin/accounts`)、商品分账规则 (`/admin/rules`)、数据同步 (`/admin/sync`)、商品口径控制 (`/admin/product-types`)
- **所在模块**: 后台写操作
- **所在元素**: 新建/保存账号、保存分账规则并重建、立即补拉、保存商品口径等
- **证据来源**: Browser 使用 4178 普通管理员角色 mock 验证：账号新建和保存可用；不分佣账号保存可用；手动补拉可用；商品口径保存可用。相同角色下，线索中心维护已禁用，线索分配试运行只读。`App.tsx` 只对后台入口做 `admin` 角色门禁，各子页并未统一接收最高管理员权限
- **是否原型 / mock 态**: 是；mock 只替换当前用户身份，控件可用/禁用由真实页面代码决定，因此可作为权限交互证据
- **现状**: 普通管理员在部分后台页仍拥有写入口，权限表现不一致
- **预期**: 按已确认方向，由 `DYDATA-32` 建立最高管理员可配置的角色权限；在该改造完成前，普通管理员目标为只读
- **修改建议**: 不在 PAGE_EXPLAINER 中定义字段级权限矩阵。由 `DYDATA-32` 统一权限源，并让各后台写按钮、表单和对应提交行为消费同一权限判断；完成后回跑普通管理员角色验收
- **解决记录**: none

### GAP-P3-02: 线索分配规则和总部池闭环尚未由专项 BRD 冻结

- **分类**: `out_of_scope`
- **所在页面**: 线索分配后台 (`/admin/clue-allocation/rules`、`/trial`、`/records`、`/headquarters`)
- **所在模块**: 固定分配策略、试运行/重建、总部线索池
- **所在元素**: 规则参数、版本发布、试运行预览与执行、总部池筛选及后续处置
- **证据来源**: Browser 已验证四个子视图和当前控件；`explainer-flow-dy-data.md` 仅锁定导航；用户已明确新分配规则、总部池流转和旧引擎退役由 `DYDATA-36` 线索中心 BRD V1.0 固化
- **是否原型 / mock 态**: 是；4175 展示演示规则、试运行和总部池记录，能证明页面交互结构，但不能证明最终生产业务规则
- **现状**: 页面已经能创建/查看规则版本、预览和执行试运行、查看记录及总部池库存，但评分、轮次、关闭入池、再投放等业务含义仍在专项 BRD 范围内
- **预期**: `DYDATA-36` 冻结线索池、跟进池、总部池、轮次、SLA、保护期、分配评分和总部池闭环后，再把相关 Phase 3 语义改为最终业务表述
- **修改建议**: 当前页面可见交互结构已由需求方确认并标记为 `locked`，但分配业务语义仍以本差异项和 `DYDATA-36` 为边界；不从 mock 数据反推最终分配规则
- **解决记录**: none

### GAP-P3-03: 结算中心正在按新业务需求由协作者改造

- **分类**: `out_of_scope`
- **所在页面**: 产品入口 (`/`)、门店榜单 (`/ranking`)、单店分账 (`/settlement`)、订单明细 (`/details`)、核销表现 (`/sales`)
- **所在模块**: 订单分佣结算中心及其下钻、导出和经营表现链路
- **所在元素**: 结算入口、榜单筛选和门店下钻、分账指标与明细下钻、订单筛选与导出、核销表现筛选与图表
- **证据来源**: 需求方于 2026-07-20 明确该模块仍由协作者持续完善和改造，且新业务需求相对当前运行版变化较大
- **是否原型 / mock 态**: 当前 PAGE_EXPLAINER 记录的是 2026-07-20 前的可运行版本，可作为历史基线，不代表改造后的最终产品语义
- **现状**: 当前页面和交互已经完成浏览器验证，但不再适合作为结算中心最终需求的锁定依据
- **预期**: 协作者版本合入后，以新版业务需求、页面和运行行为重新梳理流程 2、3 及对应逐页交互
- **修改建议**: 本轮不回环修改协作者负责的页面；相关流程和交互只作为当前版本历史基线标记为 `locked`，不得作为新版最终需求；协作者版本合入后重新执行 PAGE_EXPLAINER 并覆盖旧口径
- **解决记录**: none
