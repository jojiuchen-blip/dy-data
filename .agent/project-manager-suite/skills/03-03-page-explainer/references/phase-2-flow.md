# Phase 2: 流程梳理

> 本文件在进入 Phase 2 时由 SKILL.md 指令加载。

## 目标

以用户任务为维度，梳理跨页流程。本 Phase 仅产出流程语义文件（flow），不承担产物索引职责；索引统一由最终 Phase 的 delivery 文件收束。

## 执行步骤

1. 从 BRD 中提取所有核心用户场景
2. 对每个场景，梳理用户完成该任务需要经过的页面路径
3. 标注每个步骤对应的页面路由（必须与 page-delivery 一致）
4. 识别流程断点：是否有走不通的路径、缺失的页面、不合理的跳转

## 产物结构

文件名：`explainer-flow-<slug>.md`

模板：读取 `templates/flow.md`

## 流程图规范

- **每条流程必须有对应的 Mermaid 流程图**，表格作为补充明细
- 使用 `flowchart LR`（从左到右）布局
- 节点命名：`[页面名]` 方括号表示普通页面，`{判断条件}` 菱形表示分支，`([终点])` 圆角表示流程结束
- 箭头标注：`-->|用户动作|`，写用户做了什么（如「点击提交」「选择商品」）
- 分支场景必须画出所有路径（成功/失败/取消等业务分支）
- 节点中引用的页面名必须与 page-delivery 一致

## 注意

- 流程按用户任务组织，不按页面组织
- 每条流程必须从入口走到出口，不能有断点
- 流程中引用的路由必须与 page-delivery 一致
- 流程图和步骤明细必须一一对应，不能有流程图中有但表格中没有的页面，反之亦然
