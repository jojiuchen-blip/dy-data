# MySQL 建表与索引规范

> 来源：阿里巴巴 Java 开发手册 §5.1 建表规约 + §5.2 索引规约
> 适用：新建表、修改表结构、添加索引

---

## 建表规约

1. 【强制】表名、字段名**小写字母 + 下划线**，禁止大写，禁止数字开头。
   - 正例：`vehicle_identity`、`order_benefit`
   - 反例：`VehicleIdentity`、`orderBenefit`
2. 【强制】表名**不用复数**：`order`，不是 `orders`。
3. 【强制】布尔字段用 `is_xxx`，类型 `tinyint unsigned`（1=是，0=否）。
   - 正例：`is_active`、`is_deleted`
4. 【强制】**表必备三字段**：
   - `id`：主键，`bigint unsigned`，自增
   - `gmt_create`：创建时间，`datetime`
   - `gmt_modified`：更新时间，`datetime`
5. 【强制】小数类型用 `decimal`，禁止 `float` / `double`（精度丢失）。
6. 【强制】`varchar` 长度不超过 5000，超过用 `text` 并独立成表。
7. 【强制】长度几乎相等的字符串用 `char` 定长类型。
8. 【强制】禁用保留字作字段名（`desc`、`range`、`match` 等）。
9. 【强制】**禁止外键与级联**，外键逻辑在应用层解决。
10. 【推荐】表名格式：`业务名_表作用`，如 `badge_strategy`、`primary_label_config`。
11. 【推荐】字段含义变更时及时更新字段注释。

---

## 索引规约

12. 【强制】业务唯一字段必须建**唯一索引**。
13. 【强制】**超过 3 张表禁止 join**。join 字段类型必须一致且有索引。
14. 【强制】索引命名规范：
    - 主键索引：`pk_字段名`
    - 唯一索引：`uk_字段名`
    - 普通索引：`idx_字段名`
15. 【强制】`varchar` 字段建索引须指定长度，一般 20 字符区分度达 90%+。
16. 【强制】禁止左模糊或全模糊查询（`LIKE '%xxx'`），走搜索引擎解决。
17. 【推荐】组合索引区分度最高的字段放最左边。
18. 【推荐】利用覆盖索引避免回表。
19. 【推荐】防止字段类型不同导致隐式转换，使索引失效。
