# Phase 2: 术语表设计

> 本文件在进入 Phase 2 时由 SKILL.md 指令加载。

## 触发条件

Phase 1 完成后自动进入。

## 输入

- BRD 文件（项目类型、核心业务模型中的名词）
- 页面代码文件（标签文案、表头、枚举值展示文本中的业务名词）
- page-delivery 中的页面分组信息

## 提取逻辑

### Step 1: 从 BRD 提取业务域概念

扫描 BRD 以下区域，提取名词性业务术语：

| BRD 区域 | 提取目标 |
|----------|---------|
| 项目类型 | 行业/领域关键词 |
| 核心业务模型（§6） | 模型名称、实体名称、状态名称、角色名称 |
| 用户画像（§3.1） | 用户角色称谓 |
| 付费触发点（§7.2，若有） | 付费相关术语 |

### Step 2: 从页面代码提取 UI 业务名词

逐个读取 Vue 3 页面代码文件，提取以下位置的业务名词：

- `<template>` 中的标签文案、表头文字、按钮文案
- 枚举值的展示文本（如状态标签：草稿/已上架/已下架）
- 表单字段的 label
- 面包屑/Tab/侧边栏中的业务模块名称

### Step 3: 去重合并

- 相同含义但不同表述的术语合并为一条，选择最准确的表述
- 每条术语必须包含：**术语名** + **定义**（一句话说清楚是什么）

## 分组规则

- **全局术语**：跨页面通用的概念（如"用户"、"管理员"、项目级角色）
- **按功能域分组**：与 page-delivery 中的页面分组对齐，每个功能域一组

## 产物模板

文件名：`docs/prd/foundation/foundation-glossary-<slug>.md`

```markdown
# 术语表 - <项目名称>

> 生成时间: YYYY-MM-DD HH:MM
> 来源: foundation-builder Phase 2
> 上游: BRD-<slug> + page-delivery-<slug>

---

## 全局术语

| 术语 | 定义 |
|------|------|

---

## <功能域 A> 术语

| 术语 | 定义 |
|------|------|

---

## <功能域 B> 术语

| 术语 | 定义 |
|------|------|
```

## 增量模式

若已有前次 `docs/prd/foundation/foundation-glossary-<slug>.md`：

1. 读取前次术语表
2. 对比当前提取结果与已有术语
3. **新增**：前次不存在的术语 → 追加到对应分组
4. **修改**：定义发生变化的术语 → 更新定义
5. **可疑**：前次存在但当前未提取到的术语 → 标记为 `⚠️ 待确认`（不自动删除）
6. 向用户呈现变更项，等待确认

## 用户确认要点

向用户确认时，提示关注：

1. 是否有遗漏的业务术语
2. 术语定义是否准确
3. 分组是否合理
4. 同义词是否已合并（一个概念只应有一个术语名）

确认后方可进入 Phase 3。
