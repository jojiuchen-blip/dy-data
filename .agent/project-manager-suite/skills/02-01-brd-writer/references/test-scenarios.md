# 测试与验收场景

本文件供开发和调试时参考，运行时不需要主动读取。

## 基础流程

1. 输入"我想做个内部用的写日志工具"。
2. 预期：先确认项目类型（创新型），再苛刻诊断，再单题选项追问，不直接给终稿。
3. 输入"希望支持自动打标签和导出报表"。
4. 预期：识别功能堆砌风险（Kano 兴奋型过度），给互斥替代方案聚焦 MVP。
5. 输入"你看着办，随便"。
6. 预期：不终稿，继续追问关键字段。
7. 存在前后冲突选择。
8. 预期：先解冲突，再推进。
9. 全字段确认后。
10. 预期：一次性输出完整 BRD，章节根据项目类型裁剪，并附下游交接清单。
11. 页面类项目在终稿前仍缺少页面定位字段。
12. 预期：继续追问，不直接终稿。
13. 全字段确认并输出终稿后。
14. 预期：项目目录下出现 `BRD-*.md` 文件，且路径回报正确。

## 项目类型分支

15. 输入"帮我重构一下现有的订单后台"。
16. 预期：识别为改造型项目，跳过价值主张相关字段，聚焦改造动因、痛点、技术方案对比。度量门检查基线+目标值而非公式+周期。
17. 输入"我们需要把订单数据同步到内部 BI 看板"。
18. 预期：识别为集成型项目，跳过用户画像和竞品分析，聚焦上下游系统、接口协议、集成方案。度量门检查量化描述。
19. 输入"做一个内部的运营后台"。
20. 预期：识别为运营型项目，聚焦内部使用者角色、效率目标、页面定位（操作/配置/查看）。

## 决策台账

21. Phase A 元字段（项目类型）锁定后。
22. 预期：项目目录下出现 `ledger-state-<slug>.json` 和 `brd-ledger-<slug>.md`，§1 字段总览表中"项目类型"行为 locked，类型追加字段（及页面定位字段，若适用）已展开且全为 open。
23. Phase C/D 每轮锁定一个字段后。
24. 预期：台账 §1 对应字段状态从 open 变为 locked，值已填入，锁定轮次正确；§3 追加本轮变更记录，含需求方原话摘要。
25. Phase E 执行后。
26. 预期：台账 §4 门槛状态已更新，未通过的门槛标注原因。
27. 需求方说 `展开状态`。
28. 预期：从台账 §1 读取并输出所有 locked 字段的紧凑摘要，包含元字段（项目类型）。
29. 需求方说 `只看缺口`。
30. 预期：从台账 §1 读取并输出所有 open 字段列表。

## 跨会话恢复

31. 会话中断后新开会话，项目目录已有 `brd-ledger-<slug>.md`（部分字段 locked）。
32. 预期：读取台账恢复状态，从上次中断的 Phase 继续，不重复追问已 locked 的字段，新一轮编号从台账头部 Round（JSON 的 `current_round`，脚本在每次 lock / set-phase / rollback 成功时自动回写）+ 1 续编。
33. 跨会话恢复后需求方说 `展开状态`。
34. 预期：输出的摘要和上次会话结束时一致，无遗漏。

## 回滚

35. 连续锁定 3 轮后（每轮锁定不同的新字段），需求方说 `回滚到上一轮`。
36. 预期：台账 §1 中上一轮首次锁定的字段恢复为 open，值清空；§3 追加回滚记录，变更内容渲染为 `"<值>" → null`（JSON 变更记录中同步保存状态 locked → open）。轮次编号继续递增，不回退。
37. 某轮将已锁定字段从值 A 修改为值 B 后，需求方说 `回滚到上一轮`。
38. 预期：该字段恢复为值 A 且保持 locked（不是清空为 open）；§3 回滚记录渲染为 `"B" → "A"`（状态保持 locked）。
39. 需求方连续说两次 `回滚到上一轮`。
40. 预期：第一次回滚最后一轮，第二次回滚倒数第二轮。两次回滚各自在 §3 追加独立记录。
41. 回滚后需求方说 `只看缺口`。
42. 预期：首次锁定后被回滚的字段出现在缺口列表中；修改已锁定值后被回滚的字段不出现在缺口列表中（因为仍是 locked）。

## 轮次输出格式

43. Phase B 首轮（诊断轮）。
44. 预期：输出包含【苛评结论】【三维诊断】【本轮关键问题】【回复格式】四段，不出现【选项对比】【推荐选项】。回复格式不限于 A/B/C/D。
45. Phase B 第 2 轮（确认轮，角色识别）。
46. 预期：输出包含【判断结论】【确认问题】【回复格式】三段，不出现【苛评结论】【选项对比】。回复格式为"是/否/补充说明"。
47. Phase C 常态轮（选项决策轮）。
48. 预期：输出包含【苛评结论】【本轮关键问题】【选项对比】【推荐选项】【回复格式】五段。回复格式为 A/B/C/D 或自定义一句话。
49. Phase D.5 前提挑战（确认轮）。
50. 预期：4 个前提分 4 轮逐条确认，每轮输出【判断结论】【确认问题】【回复格式】。不在 1 轮内问完 4 个前提。

## 护栏（脚本级防误操作）

51. 对已存在同 slug 台账的项目再次执行 `init`（不带 `--force`）。
52. 预期：返回 `already_exists` 错误并退出，既有台账原样保留；加 `--force` 才允许重建（会清空全部已锁定字段与变更日志）。
53. `init --project-type` 传入六个英文 token 及对应中文类型名之外的值。
54. 预期：返回 `invalid_project_type` 错误并列出全部合法取值；传中文类型名（如 `创新型`）时归一为英文键（`innovation`）后正常建账，类型追加字段完整加载。
55. 可回滚的锁定记录已全部回滚后，需求方再说 `回滚到上一轮`。
56. 预期：返回 `nothing_to_rollback` 错误，台账不变，不得返回 success。
57. 终稿落盘（DONE）后执行 `lock` 或 `rollback`。
58. 预期：返回 `phase_done` 错误拒绝执行；先 `set-phase --phase C --round <n>` 显式重开（reopen）后方可继续收敛。
