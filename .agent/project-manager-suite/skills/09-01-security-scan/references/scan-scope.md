# Security Scan Scope

> v2.0.0（2026-05-22）：本套包面向"内部 / 本机工具"场景，scan 范围聚焦应用代码安全 + 依赖与供应链 + 敏感信息。删除原 §1 网络与暴露面、§3 认证与授权（多用户）、§4 大部分公开 API 攻击面、§7 配置与部署加固（云/K8s）、§8 CI/CD 与运行时发布面。
> 如果宿主项目实际确实对外暴露或多用户部署，由用户在执行 security-scan 时显式声明，再额外触发对应维度的检查。

## Default Full Scan Domains

### 1. 应用代码安全

- 注入、命令执行、路径穿越、反序列化风险
- XSS、SSRF、模板注入、不安全文件上传
- 危险系统调用、临时绕过校验、调试开关残留

### 2. 输入校验（API 层精简版）

- 入参校验不足导致的越界 / 类型混淆 / 注入风险
- 输出字段过度暴露（误返回内部字段、堆栈、调试信息）

> 原 §3 认证授权 / §4 公开 API 攻击面 / §7 云配置 / §8 CI/CD 等维度在 v2.0.0 删除——本套包默认场景为单/小团队内部工具，不预设公开部署或多租户授权模型。

### 3. 敏感信息与密钥

- 硬编码密钥、`.env` 泄漏、日志泄漏
- 私钥、证书、访问令牌、数据库凭证暴露
- 测试账号和临时密钥未隔离

### 4. 依赖与供应链

- 第三方依赖漏洞
- 锁文件漂移、来源不清、恶意包风险
- SBOM 缺失、制品来源不可追踪
