# YYYY年MM月 规则升级候选池

> **使用说明：**
> 本文件用于收集当月开发日志（`project-devlog`）、复盘中产生的“规则升级建议”。
> 它是复盘辅助模板，不属于 3 类全局文件本体。
> 只有当某个经验或问题表现出高频性、稳定性或可执行性时，才在此处沉淀。之后定期评估，决定是否正式升级到宿主项目的权威规则载体，例如 `project-rules.md` 或 `docs/rules/*.md`。
