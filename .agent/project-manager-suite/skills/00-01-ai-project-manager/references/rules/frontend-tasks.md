# 前端开发任务规则包

## 适用任务

- 页面改造
- 组件渲染逻辑调整
- 接口接入、状态管理、交互与弹窗改造

## 加载时机

当任务进入前端写代码阶段，必须在读取全局入口后加载本文件。

## 执行顺序

1. 先读取项目规则文件
2. 读取本文件
3. 读取当前执行计划中的前端任务
4. 读取对应需求依据、接口定义、页面说明或原型资料
5. 明确哪些由后端计算、哪些只负责渲染后再写代码

## 核心规则

### 1. 前端默认只做渲染，不补业务判断

若需求或方案已明确某项逻辑由服务端输出，前端只负责按接口数据渲染。

禁止为了“先显示出来”而在前端补状态判定、标签匹配、按钮拼装或兜底业务逻辑。

### 2. 接口契约优先于现有页面实现

改造前端时，应以当前计划和接口需求 / 方案依据为准，不以旧页面结构或历史接口字段为准。

若页面现有实现与最新契约冲突，优先回查计划与需求 / 方案依据，不得凭现状延续错误逻辑。

### 3. 配置化展示必须真实接接口

颜色、文案、图标、按钮、标签等若应来自配置表或接口返回，前端必须真实接入接口字段。

禁止在本地常量、localStorage、组件内部继续保留旧硬编码。

### 4. 组件职责单一

- 页面层：负责数据请求、状态协调、页面编排
- 组件层：负责显示与交互
- 组合式函数 / store：负责状态共享与请求封装

禁止在单个组件里同时堆业务匹配、请求拼装和复杂展示逻辑。

### 5. 前端改造前先确认后端前置是否已满足

接接口前至少确认：
- 接口路径与字段已稳定
- 配置表和依赖数据已就绪
- 当前计划中的前置关系已满足

若后端前置未完成，应先回写阻塞，不要在前端凭假数据把逻辑写死。

## 最小验收口径

至少验证：
- 页面字段与接口契约一致
- 无新增硬编码业务逻辑
- 关键配置项来自接口返回
- 切换主要分支场景时能正确渲染
- 任务状态已回写到计划文档

### 6. 可保存页面必须走真实联调自测

页面若存在加载、保存、开关切换、排序调整、增删改等与后端交互行为，在宣称完成前至少执行一次真实联调自测。

最低要求：
- 真实加载一次接口，确认 `code / message / data` 口径与页面成功判断一致，禁止沿用旧页面的成功码假设
- 至少覆盖 3 个状态：加载成功态、加载失败或异常态、空数据或默认数据态
- 对可保存页面，至少执行一次真实保存链路：修改值 -> 提交 -> 成功提示 -> 页面刷新或重新加载 -> 回显结果一致
- 若页面值最终应写入数据库或配置表，需通过接口回读或数据库回查确认保存结果，不得只看 toast 或本地 state
- 若页面依赖固定枚举、类型值、行数或默认文案，必须以当前需求依据、API 返回和真实种子数据为准，禁止沿用历史常量
- 若页面依赖枚举值、状态值、类型值、条件键等口径字段，必须额外核对这些值在需求依据、接口返回、真实数据库记录、页面展示之间一致；不能只看到“页面渲染成功”就视为通过

### 7. 前端页面改造不得跳过真实数据核对

对接接口后，必须至少核对一次：
- 前端使用的字段名与真实接口字段名一致
- 页面固定常量中的 `type / key / status / 枚举值` 与真实数据库或接口返回一致
- 页面显示出来的数据确实来自接口，而不是被本地默认值静默覆盖

若真实联调发现页面异常的根因落在开发数据库数据口径，而不是前端渲染代码本身，必须回到数据库或上游接口修正源数据，再重新执行页面验证；禁止只在前端加兼容映射后跳过源数据回查。

若任一项未核对，任务不得标记为“已自测完成”。

### 8. 资源型展示链路必须逐层验证

凡是页面展示依赖图片、图标、文件预览、跳转链接等 URL 型资源，必须按以下链路逐层验证，禁止跳层：
- 数据库或配置表中的真实值存在
- 接口真实返回该值
- 前端实际取用的是该值，而不是本地默认值或兼容值静默覆盖
- 该值在当前前端环境中可访问、可渲染、可预览
- 页面最终显示结果正确，且关键入口位（列表态、弹窗/详情态、保存后回显态）一致

补充要求：
- 不能只检查字段非空或接口 `200`，必须确认浏览器侧资源真实可用
- 若页面对旧路径、旧资源目录、历史静态文件存在兼容逻辑，必须明确核对当前显示的是接口真实资源还是前端兼容回退资源
- 若资源链路失败但页面主体仍能打开，不得判定为“联调通过”

## 禁止行为

- 禁止把服务端该做的规则计算搬到前端
- 禁止为追求短期可见效果而写临时硬编码且不回收
- 禁止未读对应需求或方案依据就直接改页面
- 禁止只跑 `npm run build` 或只看页面能打开，就宣称页面联调通过
- 禁止把“字段有值”“接口返回成功”“组件渲染未报错”误判为“图片/图标/链接等资源展示正常”
