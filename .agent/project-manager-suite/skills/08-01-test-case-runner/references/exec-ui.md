# UI 用例执行规则

> 本文件在每条 UI 用例开始执行前读取。
> UI 测试模拟真人操作。判定 PASS/FAIL 的唯一依据是**截图上能看到什么** — 截图看不到的东西，用户也看不到，不能算通过。

## 执行流程

1. **读取用例文档步骤** — 打开用例详情文档，找到当前编号对应的"步骤"和"预期结果"，逐步照做

2. **设置视口** — **首次打开前**必须设置视口为桌面常用尺寸：
   ```
   browser_resize: { width: 1440, height: 900 }
   ```
   如果项目明确是移动端页面，按用例文档另行指定。

3. **打开页面（必须人类可见）** — 优先使用人类可见的 Chrome/浏览器窗口打开目标 URL，让协作中的人类可以同步看到当前页面；只有在“人类已明确接受后台执行仅用于辅助截图取证”时，才可额外使用 Playwright/MCP 做补充操作，但**不能以非可见方式直接完成 UI 判定**。
   ```
   ${APP_URL}<用例文档指定的页面路径与参数>
   ```
   其中 `${APP_URL}` 为 `application.yml` 中 `urls.app` 的实际值（见 [env-setup.md](env-setup.md)）；页面路径与参数以用例文档写的为准。

4. **截图取证** — 使用 `browser_take_screenshot` 保存截图到：
   ```
   docs/test-case/reports/screenshots/测试验收-{业务域}/
   ```
   目录不存在则先创建。

5. **读图验证（必做）** — 截图保存后，**必须用 Read 工具读取截图文件**，通过视觉识别验证页面状态。分两关：

   **第一关：视觉审查** — 按 [visual-check.md](visual-check.md) 的检查清单，以人眼视角判断页面是否正常可用。核心问题：**一个普通用户看到这个页面，3 秒内会不会觉得"坏了"？**

   快检结论只有两种：
   - ✅ 页面渲染正常，进入第二关
   - ❌ **直接判 FAIL**，记录具体问题（如"CSS 未加载，页面全黑无样式"），无需继续逐项验证

   **第二关：逐项验证** — 快检通过后，按用例文档的"预期结果"表格，**对照截图内容**逐行验证：
   - 截图上能否看到预期元素
   - 截图上文本内容是否匹配
   - 截图上元素的**页面结构位置**是否正确：在上方/下方、在卡片内/卡片外、属于哪个区域、是否打乱原有层级
   - 截图上元素的**相对关系**是否正确：是否紧跟某模块之后、是否插入到了错误区块、是否覆盖了不该覆盖的内容
   - 截图上看不到的元素 = 不存在，直接判 FAIL

6. **交互验证**（如需要）— 先用 `browser_snapshot` 获取元素引用（ref），再用 `browser_click` 执行交互，交互后再次**截图**并**Read 截图文件**验证结果（同样走第一关+第二关）

## 关键约束

- **必须人类可见**：UI 测试默认要求“页面对人类可见”。AI 不得只在无头浏览器、后台自动化窗口、隐藏窗口里完成整条 UI 用例，然后直接宣称 PASS/FAIL。
- **必须 Read 截图**：`browser_snapshot` 返回的是 DOM 树，包含视口外不可见元素，且无法反映样式是否正确加载。只有读取实际截图图片，才能像真实用户一样判断页面是否正常。**不读截图就判 PASS 是严重失职。**
- **必须关注页面结构**：UI 验证不只检查“元素有没有出现”，还必须检查它“出现在哪里”。只要用例或 PRD 写了“位于某区域 / 在某模块下方 / 在卡片内部 / 顺序为 A→B→C”，就必须把结构和相对位置当成必检项。
- **`browser_snapshot` 的唯一用途**：获取元素 ref 值供 `browser_click` 等交互操作使用。绝不能用 snapshot 的 DOM 数据判定 PASS/FAIL。
- **交互失败 = UI 缺陷**：
  - `browser_click` 超时：说明用户也无法完成该操作，判 FAIL
  - 点击成功但值不变：**尝试一次不生效就判 FAIL**，不要反复用不同的 JS 选择器、force click 等方式重试 — 真实用户只会点一下，点不动就是 bug
  - 绝不要用 `browser_run_code`/`page.evaluate()` 换花样重试。详见 [exception-handling.md](exception-handling.md)
- **截图是证据，不是执行替代**：截图用于取证和复核，但不能替代“让人类看见真实执行页面”这一步。
