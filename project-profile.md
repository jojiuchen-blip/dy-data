# 项目画像

> 本画像用于将中途接入套包的既有项目纳入 S0.5 治理。`【用户确认】` 字段来自当前权威产品定义与仓库主入口；`【系统推断】` 只表示代码或文件结构证据，不替代业务确认。

## 1. 基本信息

- 项目名称：`【用户确认】 dy-data（抖音订单分账数据看板）`
- 项目 slug：`【主入口回写】 dy-data`
- 项目一句话目标：`【用户确认】 本仓库当前主线是面向经销商的抖音来客订单分账数据看板，重点维护订单、核销、退款、抖音号明细匹配、门店销售归属和门店核销归属之间的分账口径。`
- 当前阶段：`【主入口回写】 S0.5`
- 协作模式：`【系统推断】 业务单人 + AI执行`

## 2. 身份识别

- 身份识别口径：`【系统推断】 待按宿主项目确认`

## 3. 业务目标

- 目标使用者：`【用户确认】 门店经营人员、门店负责人、集团财务人员、集团管理人员`
- 主要问题：`【用户确认】 让门店和集团财务看到可信、可追溯、可复核的跨店核销与分账参考依据，减少人工整理和复核成本`
- 第一版核心目标：`【用户确认】 采集并分析订单、核销、退款、SKU、账号绑定与门店归属数据，形成经营分析和分账参考；系统不执行真实分账或资金动作`

## 4. 页面与任务定位

- 项目覆盖对象：`【用户确认】 汽车经销商集团及其门店的抖音来客订单经营与跨店核销场景`
- 当前页面主要给谁用：`【用户确认】 门店经营与负责人、集团财务与管理人员，以及受权后台管理员`
- 当前页面主要用途：`【系统推断】 订单与门店经营分析、分账参考复核、线索运营和后台配置管理`
- 页面定位标签：`【系统推断】 数据看板 + 业务工作台 + 管理后台`

## 5. 第一版范围

- 核心功能 1：`【用户确认】 订单、核销、退款、SKU 与账号归属数据的采集、匹配和追溯`
- 核心功能 2：`【用户确认】 本店与跨店核销关系识别，以及应收、应扣、净额分账参考计算`
- 核心功能 3：`【用户确认】 门店和集团维度的经营查看、异常定位与财务复核支持`
- 已实现扩展线索：`【系统推断】 代码中还存在销售看板、线索中心、线索分配、反馈和后台配置能力；其正式业务边界需在后续 BRD / PRD 中确认`
