## 1. 基本信息

- 项目名称：`【用户确认】 dy-data（抖音经营引擎）`
- 项目 slug：`【主入口回写】 dy-data`
- 项目一句话目标：`【用户确认】 面向汽车经销商集团及门店，统一承载抖音经营数据分析、跨店核销与分账复核、线索分配与跟进、后台运营管理，以及数据采集和生产运行。`
- 当前阶段：`【主入口回写】 S0.5`
- 协作模式：`【系统推断】 业务负责人 + AI 执行；Linear 管需求生命周期，GitHub / CI 管代码与验证事实`

## 2. 身份识别

- 身份识别口径：`【系统推断】 本地 Git user.name 识别代码执行人；Linear assignee / owner 识别任务负责人；两者不一致时分别记录`

## 3. 业务目标

- 目标使用者：`【用户确认】 门店经营人员、门店负责人、集团财务、集团管理人员及授权后台管理员`
- 主要问题：`【用户确认】 将分散的订单、核销、销售、线索和门店运营数据转为可信、可追溯、可复核的经营决策与业务执行依据，降低人工整理、分配和复核成本`
- 产品边界：`【用户确认】 系统提供经营分析、线索运营、后台配置和分账参考，不执行真实资金划拨`
- 第一版核心目标：`【用户确认】 形成覆盖经营与结算、线索运营、后台管理和数据平台的一体化经营工作台，提供可追溯的经营与分账参考，不执行真实资金动作`

## 4. 页面与任务定位

- 项目覆盖对象：`【用户确认】 汽车经销商集团及其门店的抖音经营、跨店核销、线索运营和后台管理场景`
- 当前页面主要给谁用：`【用户确认】 门店经营人员与负责人、集团财务与管理人员、授权管理员`
- 当前页面主要用途：`【用户确认】 经营与结算分析、线索分配和跟进、后台运营配置、数据同步与生产管理`
- 页面定位标签：`【系统推断】 数据看板 + 业务工作台 + 管理后台`

## 5. 第一版范围

- 经营与结算：`【用户确认】 销售看板、门店排名、月度结算、订单明细、跨店核销识别、应收应扣净额参考和异常复核`
- 线索运营：`【用户确认】 线索总览与详情、跟进记录、分配轮次、规则版本、试运行、总部线索池和门店评分`
- 后台管理：`【用户确认】 账号与门店权限、SKU 规则、非佣金归属账号、商品类型可见性、反馈、同步和线索分配配置`
- 数据平台：`【用户确认】 抖音数据采集、API、数据库、Web、worker、浏览器导出、迁移、部署和运行可靠性`
- 完成度边界：`【系统推断】 路由、接口、模型或页面存在只证明能力已实现到代码层，不代表全部业务场景已完成用户验收`
