## 产品背景与定位

### 产品是什么

抖音经营引擎的门店结算能力，把销售、核销、商品、费率规则、退款调整和月度账单组织为统一、可复核的经营与结算依据。用户打开后可以先看全国门店对标，再核对单店月度双费用，并下钻到一张券一个费用方向的计算依据。推广服务费与管理服务费独立计算和追溯，正式累计从 `2026-08` 开始，`2026-07` 测试数据不进入正式累计或开票准备范围。本产品不执行真实资金划拨，本轮也不提供在线开票或财务审核操作。

### 使用者

| 使用者 | 使用场景 |
|--------|---------|
| **总部运营人员** | 按月度或正式累计口径查看全国门店榜单，比较销售、核销和双费用表现并识别异常门店。 |
| **门店经营人员** | 查看授权门店的月度经营与费用结果，并从产品汇总下钻到订单费用依据。 |
| **门店财务人员** | 核对推广服务费应收、管理服务费应扣、退款调整、规则版本和开票准备范围。 |
| **授权后台管理员** | 维护参与分账 SKU 的商品分类、双费率和结算范围，检查商品同步状态，为四个查看型页面提供可信规则基础；本清单不新增未经页面确认的管理端页面。 |

## 区块详情

### ① 排名筛选与结果

**所属页面**：全国门店榜单（路由: #/ranking）

**布局**：

```
[月度/累计] [月份 ▼] [产品范围 ▼] [商品类型 ▼]
[门店搜索____________] [排名依据 ▼]
[销售与核销指标] [推广服务费] [管理服务费] [结算参考净额]
[名次 | 门店 | 销售 | 核销 | 双费用 | 结算参考]
```

**业务逻辑**：
- 月度口径只统计所选月份；正式累计从 `2026-08` 累计到所选截止月，`2026-07` 测试数据不得混入。
- 产品范围与商品类型使用同一筛选口径，商品类型必须受产品范围约束；门店关键词和排名依据变化后，摘要指标、门店数量与排名结果同步更新。
- 双费用指标展示调整后净额；结算参考净额为推广服务费净额减管理服务费净额，不代表真实资金划拨。
- 全国榜单前 20 名是授权 Web 用户的横向对标例外，不授予其他门店的单店账单或订单明细权限。
- 无匹配门店时显示明确空态；尚无正式累计数据时显示缺省值和正式账期说明，不沿用上一筛选结果。

**涉及使用者**：总部运营人员

### ② 费用汇总与订单下钻

**所属页面**：单店分账（路由: #/store）

**布局**：

```
[月份 ▼] [当前门店（只读）] [产品范围 ▼] [商品类型 ▼]
[销售/核销指标] [推广服务费净额] [管理服务费净额]
[推广服务费汇总行........................查看订单]
[管理服务费汇总行........................查看订单]
```

**业务逻辑**：
- 当前门店由用户的数据权限范围决定，门店经营和财务人员不能借助页面参数切换到未授权门店。
- 同一月份和产品筛选下，同时展示销售、核销、推广服务费应收和管理服务费应扣；每个费用方向保留原始金额、调整金额和调整后净额。
- 退款不覆盖原费用结果：原始结果保留在原始发生月份，负向调整计入退款发生的调整入账月份；部分退款按退款后净额同比例减少两个费用方向的计算基数。
- 日级费率规则可能使同一汇总行包含多个实际费率和规则版本，页面应如实表达费率范围或集合，不把汇总行伪装为单一费率。
- “查看订单”携带账期、门店、产品维度、费用方向和实际规则上下文，订单页恢复来源口径；上下文仅用于筛选，进入明细时仍需重新校验权限和真实结果。
- 未配置费率、没有有效订单或来源口径已变化时，分别给出可理解的原因、空态和返回汇总页路径。

**涉及使用者**：门店经营人员、门店财务人员

### ③ 费用类型、筛选与明细表

**所属页面**：订单费用明细（路由: #/orders）

**布局**：

```
[推广服务费明细] [管理服务费明细]              [导出当前明细]
[业务月份 ▼] [销售月份 ▼] [核销月份 ▼] [数据状态 ▼]
[订单 ID、券 ID、SKU 或商品名称________________]
[一张券一个费用方向：基数 | 费率 | 金额 | 调整 | 版本]
```

**业务逻辑**：
- 从单店分账进入时恢复服务端核验后的账单、门店、月份、产品维度、费用方向、费率和规则版本上下文，并定位到核对区域；缺少上下文时使用有说明的默认筛选。
- 推广服务费与管理服务费始终单选；推广服务费按销售业务日匹配规则并归入销售月份，管理服务费按核销业务日匹配规则并归入核销月份。
- 明细以一张券的一个费用方向为一行，展示原始发生月份、规则匹配日、费用基数、实际费率、原始费用、调整金额、调整后净额和不可变规则版本。
- 退款或取消核销通过独立调整记录展示，同时保留原始发生月份和调整入账月份；已锁账结果不因后续费率修改而重算或覆盖。
- 列表和导出使用相同筛选、权限和费用口径；导出前重新校验权限，并保留规则版本、原始发生月份和调整入账月份等复核信息。
- 非法或过期上下文不能参与计算；所选方向无数据时显示方向性空态，空结果不允许导出，导出失败可重试且不丢失筛选条件。

**涉及使用者**：门店经营人员、门店财务人员

### ④ 开票流程引导

**所属页面**：开票确认（路由: #/invoice）

**布局**：

```
[当前功能暂未开放]
[01 月度账单待确认] [02 门店确认账单]
[03 门店开具推广服务费发票] [04 财务审核]
[05 审核完成并进入结算]
[前置条件] [需准备材料] [预计开票范围] [支持渠道] [常见问题]
```

**业务逻辑**：
- 页面只读展示从月度账单生成、门店确认、开具推广服务费发票、财务审核到进入结算的五个顺序节点，不提交发票或改变账单状态。
- 页面必须持续显示“当前功能暂未开放”，不得出现会被误认为可以在线开票、确认账单或提交财务审核的主操作。
- 预计开票范围只包含正式账期已确认账单中的推广服务费；`2026-07` 测试数据和应扣管理服务费不计入门店推广服务费开票金额。
- 前置条件、需准备材料、预计范围、支持渠道和常见问题用于帮助门店提前核对；账单差异应返回单店分账和订单费用明细追溯。
- 发票抬头、税号、类目、税率、接收方式和正式支持入口在财务确认前统一标记“待财务确认”或“待上线通知”，不得写成既定事实或伪造联系方式。

**涉及使用者**：门店经营人员、门店财务人员
