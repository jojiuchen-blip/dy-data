## 全局术语

| 术语 | 定义 |
|------|------|
| 抖音经营引擎 | 面向汽车经销商集团的经营、结算、线索和后台治理系统；本轮只涉及经营与结算相关能力。 |
| 经营与结算依据 | 系统根据订单、核销、商品和规则生成的可复核参考结果，不代表系统执行了真实资金划拨。 |
| 授权用户 | 已通过系统身份认证，并被授予角色、组织和数据范围的用户。 |
| 数据权限范围 | 用户可查看或操作的数据集合，可为单店、多店、区域或全国；权限必须由服务端执行，不能由 URL 参数授予。 |
| 全国榜单前 20 例外 | 向已授权 Web 用户展示全国门店排行榜前 20 名的横向对标例外；不扩展为查看其他全国明细的权限。 |
| 总部运营人员 | 在授权范围内查看全国或区域经营指标、门店排名和异常的管理角色。 |
| 门店经营人员 | 查看本店经营、结算和订单依据的经销商抖音负责人或其他授权人员。 |
| 门店财务人员 | 核对本店账单、费用方向、订单依据和开票准备信息的授权人员。 |
| 授权后台管理员 | 维护 SKU 规则、商品同步、费率规则和运行状态的后台配置角色。 |
| 正式账期 | 可进入正式费用计算、对账和开票准备的业务月份；当前明确从 `2026-08` 开始。 |
| 测试账期 | 仅用于试算和验证、不得进入正式累计或开票的业务月份；当前 `2026-07` 属于测试账期。 |
| 内部主键 (`id`) | 本系统为一行数据生成的自增数值标识，仅用于数据库主键和内部高效关联，不作为抖音接口、对账或导出的业务编号。 |
| 平台业务 ID | 抖音提供的订单 ID、券 ID 等外部稳定标识；迁移后仍保持非空唯一，用于采集幂等、平台交互、对账、导出和财务追溯。 |
| 兼容迁移 | 先新增并回填内部主键、保持旧读写可用，再切换应用关联和数据库约束的分阶段迁移方式；每阶段必须可验证并保留恢复路径。 |

## 商品与 SKU 治理术语

| 术语 | 定义 |
|------|------|
| 商品 | 抖音侧可售卖的商品主实体，具有商品 ID、SPU、名称、状态、归属账号等平台属性。 |
| 商品 ID (`product_id`) | 抖音商品主实体的稳定平台标识；与 SKU ID 不可互换。 |
| SPU (`spu_id`) | 用于归并同一标准产品的一组商品或 SKU 的平台标识；平台未返回时可为空。 |
| SKU | 商品下的具体销售规格，是费率规则和产品分类配置的最小稳定单元。 |
| SKU ID (`sku_id`) | SKU 的稳定唯一标识，是 `dim_sku_product_rules` 和费率规则关联的业务键。 |
| SKU 名称 (`sku_name`) | 面向管理员展示的 SKU 文本名称；不得代替 SKU ID 作为唯一关联键。 |
| 产品范围 (`product_scope`) | SKU 的一级业务分类，例如“养车套餐”；是商品类型的上级维度。 |
| 商品类型 (`product_type`) | 产品范围下的二级业务分类，例如“精保养套餐”；同名类型仍应结合产品范围解释。 |
| 统一 SKU 商品规则 | 以 `dim_sku_product_rules` 为唯一事实源维护 SKU ID、名称、产品范围和商品类型的规则；CSV 不再作为运行时事实源。 |
| 商品归属账号 | 平台认定的商品所有或经营主体账号，用于判断商品是否属于目标业务主体。 |
| 商品归属账号 ID (`owner_account_id`) | 商品归属账号的稳定标识，是结算筛选的权威字段；账号名称只用于展示与人工核对。 |
| 商品归属账号名称 (`owner_account_name`) | 商品归属账号的可读名称，可能发生变更，不得单独作为长期关联键。 |
| 商品创建者账号 | 创建商品记录的平台账号；与商品归属账号是不同概念，不得混用。 |
| 商品状态 | 商品在抖音侧的在线、下架、删除或其他生命周期状态。 |
| 有效商品 | 满足结算纳入条件的商品；至少要求平台状态有效，并符合目标商品归属账号和渠道规则。 |
| 平台同步字段 | 从抖音商品在线 API 获取并由同步任务维护的商品、SPU、创建者、归属账号、状态和同步时间等字段。 |
| 人工维护字段 | 由后台管理员维护、商品同步不得覆盖的产品范围、商品类型和费率等业务规则字段。 |
| 非分佣归属账号 | 历史规则中用于排除特定商品归属账号的名单；新双费用模型仍需明确其是否继续生效，不能默认为目标商品归属账号的反义词。 |

## 费率规则术语

| 术语 | 定义 |
|------|------|
| SKU 费率规则 | 针对一个 SKU 维护推广服务费率、管理服务费率、生效日、状态和版本的业务规则。 |
| 推广服务费率 (`promotion_service_fee_rate`) | 计算销售门店应收推广服务费所使用的比例，存储为 `0` 到 `1` 的小数。 |
| 管理服务费率 (`management_service_fee_rate`) | 计算核销门店应扣管理服务费所使用的比例，存储为 `0` 到 `1` 的小数。 |
| 同费率开关 (`rates_equal`) | 管理界面中的便捷输入状态；开启时两个费率保存为同值，关闭时允许分别输入，不作为独立财务结果。 |
| 规则生效日 (`effective_date`) | 规则开始参与订单业务日匹配的自然日期；首批固定为 `2026-08-01`，后续单条维护或导入批次可选择到日，统一按 `Asia/Shanghai` 当日 `00:00:00` 生效。 |
| 规则状态 | 费率规则当前是否允许被新计算匹配，建议限定为“启用 / 停用”；历史订单仍保留其已使用版本。 |
| 规则版本 (`rule_version`) | 一次已发布费率规则快照的不可变标识，用于订单结果追溯、导出和差异核对。 |
| 不可变历史 | 已生效或已被计算引用的规则版本不得原地改写；修改通过创建新版本表达。 |
| 当前规则 | 对某个 SKU 和费用方向，在指定订单业务日不晚于该日的规则中，生效日期最新且状态有效的不可变版本。 |
| 锁账 (`statement_locked`) | 某账期的费用结果已确认并冻结；后续费率修改不重算、不覆盖已锁账结果，退款等后续业务事件通过独立调整记录体现。 |
| 批量导入批次 | 一次参与分账 SKU 费率模板的上传、预校验、确认写入和结果输出审计单元；模板行的业务字段固定为 SKU 名称、SKU ID、推广服务费率和管理服务费率，规则生效日由批次统一选择。 |
| 导入预校验 | 在实际写入前对模板格式、四个必填字段、SKU 名称与 SKU ID 对应关系、双费率范围及重复规则进行全量验证；错误必须定位到原文件行号、字段和可操作原因。 |
| 全量原子写入 | 批量导入只有在全部行预校验通过后才一次性发布规则；任一行非法时整批不写入正式费率规则，但保留批次、逐行错误和结果文件。 |
| 导入结果文件 | 保留四个原始业务字段，并按原始行返回校验结果、错误字段、错误原因和成功时生成的规则版本；同一行存在多个错误时全部列出。 |
| 批量操作审计 | 记录操作人、时间、文件摘要、校验结果、写入结果和失败原因的后台留痕。 |
| 重复规则 | 同一 SKU 在同一规则生效日只能存在一个规则版本，重复时拒绝写入；同一月份允许存在多个不同生效日的不可变版本，订单按业务日匹配当时最新有效版本，历史结果和已锁账结果不受后续版本影响。 |

## 双费用结算术语

| 术语 | 定义 |
|------|------|
| 销售门店 | 通过抖音直播或短视频促成订单销售的门店，是推广服务费的应收方。 |
| 核销门店 | 实际完成券或订单核销服务的门店，是管理服务费的应扣方。 |
| 销售业务日 (`sale_business_date`) | 销售时间按 `Asia/Shanghai` 转换得到的自然日，用于匹配推广服务费率。 |
| 核销业务日 (`verify_business_date`) | 核销时间按 `Asia/Shanghai` 转换得到的自然日，用于匹配管理服务费率。 |
| 销售月份 (`sale_month`) | 按销售时间归属的 `YYYY-MM` 业务月份，用于汇总推广服务费；费率匹配使用销售业务日。 |
| 核销月份 (`verify_month`) | 按核销时间归属的 `YYYY-MM` 业务月份，用于汇总管理服务费；费率匹配使用核销业务日。 |
| 抖音销售渠道 | 订单的抖音内容成交来源；本轮正式纳入范围仅包括直播和短视频。 |
| 推广服务费 | 销售门店因通过直播或短视频卖出有效商品、且订单进入可结算状态而应收的费用；按销售业务日匹配费率，归入销售月份。 |
| 推广费 | “推广服务费”的页面简称；API、Schema、导出表头和财务口径应优先使用完整名称，简称不得产生第二种计算语义。 |
| 厂端激励推广费 | 全国榜单原型中的历史页面文案，在本轮 Foundation 中映射为推广服务费调整后净额；生产文案应统一为“推广服务费”，不另建费用方向。 |
| 管理服务费 | 核销门店因完成直播或短视频渠道有效商品订单核销而应扣的费用；按核销业务日匹配费率，归入核销月份。 |
| 双费用模型 | 将推广服务费和管理服务费作为两个独立费用方向计算、展示、下钻和追溯的结算模型。 |
| 分佣 (`commission`) | 现有代码对费用计算的历史统称；新契约和用户文案应优先使用“推广服务费 / 管理服务费”，避免一个 `commission_rate` 同时代表两个方向。 |
| 费用计算基数 | 费率乘算所使用的方向性净额：推广服务费使用销售净额，管理服务费使用有效核销净额；部分退款按退款后净额同比例减少两个方向的基数，全额退款归零，未支付关闭订单不纳入。 |
| 费用方向 (`fee_direction`) | 标识一条结果属于推广服务费还是管理服务费；API 枚举为 `PROMOTION / MANAGEMENT`，Schema 存储映射为 `1 / 2`。 |
| 订单费用结果 | 一张券在指定费用方向下的不可变计算快照，包含原始发生月份、基数、费率、金额、规则版本和状态；后续退款不改写原快照。 |
| 月度结算账单 (`settlement_statement`) | 以“门店 + 账单月份”为唯一范围的账单头，保存双费用汇总、确认状态和锁账信息；产品维度不拆成多个账单头。 |
| 账单汇总行 (`settlement_statement_line`) | 月度结算账单内按“费用方向 + 产品范围 + 商品类型”形成的锁定汇总，分别保存原始金额、调整金额和调整后净额。 |
| 账单来源项 (`settlement_statement_entry`) | 月度结算账单实际纳入的一条不可变依据，精确引用某个订单费用结果或费用调整，并快照订单、券、产品维度、金额和规则版本。 |
| 跨店订单 | 销售门店与核销门店不同的订单；两个费用方向分别归属，不得用单一门店字段覆盖。 |
| 退款调整 | 退款或部分退款发生后生成的独立负向费用记录，关联原订单费用结果和原规则版本；原结果保留，调整计入退款发生月份。 |
| 原始发生月份 (`original_business_month`) | 原订单费用结果所属月份；推广服务费取销售月份，管理服务费取核销月份。 |
| 调整入账月份 (`adjustment_posting_month`) | 退款或取消核销调整记录计入报表的月份，取调整事件发生月份，不回写原始发生月份。 |
| 调整后净额 (`adjusted_net_fee_cent`) | 原始费用金额与截至当前的全部调整金额之和；报表同时保留原始金额和调整金额，不能只展示覆盖后的单值。 |
| 延迟核销 | 销售月份与核销月份不同的订单；推广服务费归销售月，管理服务费归核销月。 |
| 正式累计 | 从 `2026-08` 正式账期开始累计的指标，不包含 `2026-07` 测试数据。 |

## 商品同步术语

| 术语 | 定义 |
|------|------|
| 抖音商品在线 API | 用于获取商品、SKU、SPU、账号归属、状态等最新平台属性的外部接口集合。 |
| 商品同步任务 | 按配置或人工触发调用抖音商品在线 API，并将平台同步字段写入本地的后台任务。 |
| 当前商品快照 | 每个商品或 SKU 最近一次成功同步后的平台属性，用于当前查询和结算筛选。 |
| 商品同步历史 | 每次同步观察到的平台属性快照与差异记录，用于追溯商品状态和归属变化。 |
| 同步游标 | 外部 API 分页或增量拉取的位置标记，必须在成功提交对应数据后推进。 |
| 最近成功同步时间 | 商品同步任务最近一次完整成功写入的时间，不得用任务启动时间代替。 |
| 同步失败 | 外部请求、鉴权、限流、数据校验或写入失败导致当前批次未完整完成的状态；失败不得覆盖人工维护字段。 |
| 脱敏响应样例 | 隐去账号、凭证和真实业务数据后保留字段结构的外部 API 响应，用于确认 DYDATA-30 映射契约。 |

## 页面与报表术语

| 术语 | 定义 |
|------|------|
| 全国门店榜单 | 按月度或正式累计口径比较授权范围内门店销售、核销和结算表现的页面。 |
| 月度口径 | 仅统计所选 `YYYY-MM` 业务月的指标。 |
| 累计口径 | 从正式账期开始到所选截止月的指标合计。 |
| 单店分账 | 展示授权门店在指定月份和产品维度下的推广服务费应收、管理服务费应扣及订单依据的页面。 |
| 订单费用明细 | 以一张券或订单一行展示费用方向、基数、费率、金额、状态和规则版本的核对页面。 |
| 来源上下文 | 从汇总页进入明细页时携带的月份、门店、产品维度、费用方向、实际费率集合和规则版本集合；日级规则下可能同时有多个费率与版本，仅用于恢复筛选，不授予权限。 |
| 开票确认 | 展示账单确认、推广费开票准备、财务审核与进入结算路径的只读引导页。 |
| 开票准备指引 | 开票确认页中的前置条件、材料、预计范围、支持渠道和 FAQ；不等同于在线开票能力。 |
| 预计开票范围 | 当前指门店已确认账单中的推广服务费金额；管理服务费为应扣费用，不计入本页所述的门店推广费开票金额。 |
