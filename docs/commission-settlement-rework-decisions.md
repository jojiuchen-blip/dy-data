# 分佣与月度结算改造决策记录

更新日期：2026-07-10
当前阶段：业务规则讨论与 Mock 确认，尚未修改生产代码

本文档记录全国门店榜单、单店月度分账看板、订单费用明细、后台商品费用规则、月度锁账及门店开票管理的已确认规则和待确认事项。

## 1. 全局计算规则

### 已确认

- 所有月份划分统一使用北京时间 `Asia/Shanghai`。
- 金额以“分”为存储和计算单位。
- 每张有效核销券分别计算费用，乘比例后四舍五入到分，再进行汇总。
- 已退款、已关闭、已撤销的券明细不进入核销金额和费用计算。
- 一单多券且部分券退款时，只排除退款券，其余有效券继续参与核销金额和费用计算。
- 券级核销金额为空时标记为数据异常，不回退使用订单整单金额，也不计算推广费或服务费。
- 推广费比例和服务费比例为空表示待配置，不等同于 `0%`。
- 比例明确配置为 `0%` 表示已配置：对应金额进入统计，费用结果为零。

## 2. 全国门店销售情况榜单

Mock：[store-ranking-mock.html](./store-ranking-mock.html)

### 展示字段

1. 销售订单数量
2. 销售订单金额
3. 核销收入
4. 厂端激励推广费

本页不展示厂端应收服务费，也不展示原“有效分佣收入”。

### 计算口径

- 销售订单数量：按销售归属门店和销售时间统计，包含未核销订单；排除无效明细后，每个 `order_id` 只计一次。
- 销售订单金额：与销售订单数量使用同一批有效订单；每个 `order_id` 取订单表整单实收金额一次。
- 一单多券不会重复累计整单金额。
- 部分券退款但订单仍有有效券时，销售订单金额仍取订单表记录的整单实收金额。正式开发必须增加专项测试，避免被误认为退款后净额。
- 核销收入：按实际核销门店和核销时间汇总有效核销券级实收金额。
- 厂端激励推广费：按销售归属门店和核销时间归集有效核销券，逐券计算“券级实收金额 × 推广费比例”。
- 榜单全部指标只统计已配置推广费比例的商品类型。

### 排名规则

- 默认按销售订单数量降序。
- 数量相同时按门店 ID 固定顺序排列，保证重复查询结果稳定。

## 3. 单店月度分账看板

Mock：[single-store-monthly-settlement-mock.html](./single-store-monthly-settlement-mock.html)

### 页面名称

`单店月度分账看板`

### 筛选项

- 月份
- 门店
- 产品范围
- 商品类型

### 顶部指标顺序

1. 销售总金额
2. 本店卖出订单已核销金额
3. 应收推广费
4. 本店核销金额
5. 应扣服务费

### 指标口径

- 销售总金额：当前门店作为销售归属门店，按销售时间统计，包含未核销订单；按 `order_id` 去重后取订单表整单实收金额。
- 本店卖出订单已核销金额：当前门店卖出的订单在所选核销月份于本店或其他门店完成有效核销的券级实收金额，是推广费计算基数。
- 应收推广费：本店作为销售归属门店，逐券计算有效核销券级实收金额乘商品类型推广费比例。
- 本店核销金额：本店作为实际核销门店，按核销时间汇总有效核销券级实收金额，不限制销售归属门店。
- 应扣服务费：本店作为实际核销门店，逐券计算有效核销券级实收金额乘商品类型服务费比例。
- 同店销售并核销时，同一张有效券可以同时产生应收推广费和应扣服务费。
- 新逻辑不再使用 `relation_type=cross_store` 或 `relation_type=same_store` 作为统计条件。

### 明细表

仅保留两张表。

#### 应收推广费用：本店卖出的订单已被核销

- 商品类型
- 对应核销单数
- 对应订单实收金额
- 对应推广费比例
- 预计应收推广费

筛选条件为销售归属门店等于当前门店、按核销月份统计，不限制实际核销门店。

#### 应扣服务费用：进入本店完成核销的订单

- 商品类型
- 对应核销单数
- 订单实收金额
- 对应服务费扣除比例
- 预计应扣服务费

筛选条件为实际核销门店等于当前门店、按核销月份统计，不限制销售归属门店。

“对应核销单数”实际按有效核销券计数，每个 `coupon_id` 计一次，不是去重订单数。

### 待最终确认

- 销售总金额是否统计产品范围内所有有效商品，不要求配置任一费用比例；建议不受费用比例配置影响。
- 两张明细表是否提供跳转订单费用明细入口，以及跳转携带的筛选条件。

## 5. 后台商品费用规则

### 配置结构

配置拆成两层：

1. SKU 映射层：`SKU → 产品范围 → 商品类型`
2. 商品类型费用规则层：`产品范围 + 商品类型 → 推广费比例 + 服务费比例`

同一产品范围、同一商品类型的所有 SKU 自动继承相同比例，不允许逐 SKU 设置不同费率。

### 删除旧逻辑

- 删除“订单归属账号不分佣”名单。
- 删除“是否参与分账”开关。
- 删除同店／跨店分佣限制。
- 删除单一 `commission_rate` 作为新业务费用规则的设计。

### 强限制

- 产品范围和商品类型必填，并从有效选项中选择，不能自由创建不受控值。
- 同一 SKU 在同一生效期间只能归属一个产品范围和商品类型。
- 同一“产品范围＋商品类型”不能存在时间重叠的有效规则。
- 推广费比例和服务费比例只能为 `0%～100%`，最多两位小数。
- 非法比例输入必须阻止保存，不能静默转换为 `0%`。
- 已锁账月份使用的规则不能直接修改。
- SKU 归类调整必须选择生效月份并保留历史版本。

### 两项比例关系

- 新建规则时推广费比例和服务费比例默认相同。
- 用户修改其中一项后，两项独立保存，不能自动覆盖另一项。
- 服务费比例低于推广费比例时显示醒目提醒，但不禁止保存。

### 权限

- 厂端管理员维护 SKU 归类、费用比例和生效月份。
- 店端账号只能查看规则，不能修改。

## 6. 月度锁账

### 时间规则

- 月末后第 1～3 个自然日为金额浮动期。
- 浮动期内处理退款、数据同步、券级金额异常和规则遗漏，并持续提醒厂端处理。
- 浮动期结束后开放店端主动锁账。
- 月末后第 5 个工作日结束时仍未主动锁账，系统视为门店默认同意并自动锁账。
- 工作日按中国法定工作日历计算。

### 锁账含义

- 店端主动锁账即金额确认，不设置重复的独立确认环节。
- 锁账按“门店＋月份”独立执行。
- 底层保留不可变账单版本，页面显示当前状态、数据截止时间和最后更新时间，不必突出显示 `V1/V2`。
- 门店提交锁账时，后端必须校验页面账单版本；金额已更新时拒绝旧版本锁账。

### 锁账记录

- 锁账方式：门店主动／系统自动
- 所属门店
- 主动锁账的店端账号
- 锁账时间
- 锁账时应收推广费
- 锁账时应扣服务费
- 账单数据版本

系统自动锁账时记录“到期未操作，系统按默认同意锁账”。

### 自动锁账条件

- 三天金额浮动期已结束。
- 当月数据同步完成。
- 没有未处理的金额异常。
- 有效核销券的 SKU 归类完整。
- 应用到有效核销券的推广费比例和服务费比例已配置。
- 门店没有提交异议或延期申请。
- 自动执行前账单金额没有再次变化。
- 当前账单尚未锁账。

不满足条件的门店进入“锁账异常”列表，不能自动锁账。

### 锁账后流程

- 锁账后进入门店向厂端开票阶段。
- 厂端向门店开具的服务费发票不纳入本系统的门店发票维护模块。

### 待后续讨论

- 锁账后的退款、撤销或数据修复如何形成后续账期调整。
- 特殊情况下的解锁权限、原因和审计要求。

## 7. 门店向厂端开票

### 页面位置

- 单店月度分账看板新增“月度确认与开票”模块。
- SAP 编码属于长期资料，建议放在“门店账号设置 → 开票资料”。
- 月度看板只读展示 SAP 编码并提供维护入口，不要求每月重复填写。
- 厂端新增“后台管理 → 门店开票台账”，用于查看和导出全国数据。

### 收票方信息

仓库只保存示例占位值；正式收票方信息由后台安全配置，不写入代码或文档：

- 名称：示例汽车销售有限公司
- 纳税人识别号：91440000XXXXXXXXXX
- 地址：示例市示例区示例路 100 号
- 电话：0755-00000000
- 开户行：示例银行示例支行
- 开户账号：00000000000000000

### 发票开票信息

- 发票类型：数电专票
- 项目名称：推广服务费
- 税收分类编码：3079900000000000000
- 商品和服务分类简称：生产生活服务
- 税率：6%

以上内容作为后台可维护的厂端开票配置，不直接写死在前端。页面为各字段提供复制按钮和醒目提示。参考发票的具体摆放区域待设计时确定。

### 基础数据

建立门店开票账户数据，记录：

- 门店 ID
- 门店名称
- 门店账户认证主体
- SAP 编码
- 生效时间和失效时间
- 修改店端账号和修改时间

认证主体或 SAP 编码发生变化时新增版本，不覆盖旧值。锁账时保存当月认证主体和 SAP 编码快照。

每月锁账后写入月度待开票金额：

- 账期
- 门店 ID
- 认证主体快照
- SAP 编码快照
- 本店卖出订单已核销金额
- 锁账应收推广费
- 锁账时间
- 开票状态
- 当前有效发票 ID

### 累计金额

- 门店未开票金额：未关联当前有效发票的月度锁账应收推广费合计。
- 门店已开票金额：已关联当前有效发票的月度锁账应收推广费合计。
- 发票被替换后，旧发票退出当前有效统计，但历史记录永久保留。

### 多账期开一张票

- 一张数电专票可以覆盖多个完整账期。
- 一个账期必须一次性全额加入一张发票，不允许拆分到多张发票。
- 已被草稿以外的待处理发票占用的账期不能重复选择。
- 底层必须建立“发票－账期关联”，记录每张发票覆盖的门店、月份和金额。

### 发票替换

- 同一门店、同一月份只允许存在一张当前有效蓝字发票。
- 发票开错后，原记录不能删除。
- 门店使用“替换发票”操作，选择原因：系统录入错误／已红冲／已作废／其他。
- 新发票通过强校验后成为当前有效发票。
- 原发票状态变为“已被替换”，并通过替换关系关联新发票。
- 本系统不使用“蓝字金额减红字金额”计算顶部已开票金额，只统计当前有效发票关联的账期金额。

### 发票回填字段

- SAP 编码（从长期资料带出）
- 发票号码
- 开票日期
- 不含税金额
- 税额
- 价税合计
- 备注（可选）

发票类型固定为数电专票，税率固定为 `6%`，门店不能修改。

### 已确认的强校验

- SAP 编码已经维护，并与所选账期的 SAP 编码一致。
- 所选账期全部已经锁账，且尚未关联其他有效发票。
- 数电专票号码必须为 20 位纯数字。
- 发票号码不能被其他门店或其他账期重复使用。
- 发票号码前两位必须与开票年份一致。
- 开票日期不得早于锁账日期，也不得晚于当前日期。
- 税率固定为 `6%`。
- 发票价税合计必须等于所选账期应收推广费合计。
- 不含税金额加税额必须等于价税合计。
- 一个账期必须全额开票，不允许部分分配。
- 替换发票必须填写替换原因，新旧发票号码不能相同。
- 草稿可以暂存不完整内容；所有校验通过后才允许最终提交。

### 建议数据表

- `store_billing_accounts`：门店认证主体、SAP 编码及版本。
- `store_monthly_invoice_balances`：每月锁账金额和开票状态。
- `store_invoices`：当前及历史发票信息、替换关系。
- `store_invoice_period_allocations`：发票与门店、账期、金额的关联。

实际表名可在数据库迁移设计阶段结合现有命名规范调整。

## 8. 待确认事项

6. 门店长期 SAP 编码维护页面的完整交互和厂端复核方式。
7. 锁账后退款、撤销和金额调整的处理机制。
8. 单店看板两张费用表跳转订单费用明细的入口和筛选参数。
