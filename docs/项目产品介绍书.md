# 抖音订单分账数据看板产品定义

## 一、产品定位

本产品是面向汽车经销商集团和门店的抖音来客订单分账数据看板。系统采集并分析抖音来客订单、券核销、退款、商品 SKU、抖音号/职人绑定和门店归属等数据，识别销售归属门店与实际核销门店之间的关系，帮助门店了解自己的跨店核销和分账参考情况，并为集团财务复核门店间分账参考金额提供数据支持。

本产品不在系统侧执行交易、核销、退款、开票、付款、打款或真实分账动作。系统输出的是经营分析结果和分账参考数据，不是财务记账凭证，也不是资金划拨指令。

## 二、核心业务背景

抖音来客服务产品存在“A 店带货、B 店核销”的业务场景。订单销售归属、实际服务门店和后续退款状态可能不在同一个门店维度上。集团需要基于底层订单、核销和退款数据，识别跨店核销关系，并按照约定比例形成可供门店查看和财务复核的参考金额。

因此，本产品要解决的不是“如何完成分账动作”，而是“如何让门店和集团财务看到可信、可追溯、可复核的分账参考依据”。

## 三、目标用户

- 门店经营人员：查看本店抖音订单销售归属、核销归属、退款和跨店核销情况。
- 门店负责人：了解本店带货被他店核销、他店带货在本店核销所带来的应收、应扣和净额参考。
- 集团财务人员：获取跨店核销、退款冲减、异常记录和分账参考汇总，减少人工整理 Excel 的工作量。
- 集团管理人员：按月份、门店、商品类型观察经营结果和跨店服务结构。

## 四、核心数据来源

当前数据主要来自以下来源：

- 抖音来客订单数据：订单 ID、商品 SKU、下单时间、订单状态、券状态、实付金额、退款金额等。
- 券核销记录：券 ID、核销 ID、核销时间、核销门店、核销状态、核销类型等。
- 退款数据：订单 ID、退款状态、退款完成时间、退款金额、退款原因等。
- 抖音号/职人绑定数据：订单归属人、绑定门店、商家主体、绑定状态等。
- SKU 映射配置：将抖音 SKU 归类为业务上使用的商品类型。
- 门店和账号映射数据：用于解释销售门店、核销门店和认证主体之间的关系。

## 五、核心业务口径

### 1. 销售归属口径

销售归属数据是分账参考计算的输入，用于解释订单由哪个门店或抖音号带货产生。当前核心口径包括：

- 按订单 ID 去重，避免重复统计。
- 剔除支付取消订单后统计有效下单量。
- 根据券状态识别待核销、已核销和已退款。
- 按 SKU 映射商品类型，用于分账看板按商品类型展示和筛选。
- 结合订单归属人、抖音号/职人绑定和后台账号映射识别销售归属门店。

销售数据不作为当前主线的独立销售看板，而是作为分账基础表、跨店核销判断和财务复核的底层数据来源。

### 2. 跨店核销口径

跨店核销用于识别销售归属门店和实际核销门店不一致的情况：

- 销售门店来自订单归属人、抖音号/职人绑定或后台账号映射。
- 核销门店来自券核销记录。
- 当销售门店与核销门店一致时，记为本店自销自核。
- 当销售门店与核销门店不一致时，记为跨店核销。

### 3. 分账参考口径

分账参考数据用于门店查看和集团财务复核：

- 本店卖出、他店核销：形成销售门店的应收分佣参考。
- 他店卖出、本店核销：形成核销门店的应扣分佣参考。
- 净分佣参考金额 = 应收分佣参考金额 - 应扣分佣参考金额。
- 分佣比例来自配置或当前业务约定。

这些金额只表示“根据当前数据和口径计算出的参考值”。最终财务确认、实际付款或结算动作不在本系统内完成。

### 4. 异常与复核口径

系统需要识别影响分账参考可信度的异常情况，包括：

- 核销券未匹配到订单。
- 订单归属人未匹配到抖音号/职人绑定信息。
- 核销门店缺失或无法匹配。
- 订单已退款或存在退款冲减。
- 销售门店、核销门店、认证主体之间存在不一致，需要人工复核。

异常记录的目标是辅助财务和运营定位问题，不代表系统自动完成纠错或审批。

## 六、核心功能

### 1. 分账基础数据采集与匹配

用于形成可复核的分账基础数据：

- 导出抖音来客订单、券核销和退款数据。
- 匹配订单归属人、后台抖音号、职人绑定、门店 POI 和核销门店。
- 按核销月份生成分账基础表。
- 保留订单、券、退款、门店、SKU 等关键字段，支持后续追溯。

### 2. 门店分账参考看板

用于查看跨店核销带来的门店间参考分佣：

- 本店卖出、他店核销明细。
- 他店卖出、本店核销明细。
- 门店应收分佣参考、应扣分佣参考和净额参考。
- 按月份、门店、商品类型筛选。
- 门店净额参考排行和明细追溯。

### 3. 数据导出与复核支持

用于支撑集团财务和运营复核：

- 输出分账基础表。
- 输出异常名单和诊断明细。
- 保留订单、券、退款、门店、SKU 等关键字段，便于追溯到原始数据。

### 4. 页面 1：全国门店销售情况榜单

页面用于从集团视角查看全国门店在指定时间范围内的销售、核销和分账相关表现。页面支持筛选全部产品或某一类特定服务产品。榜单只保留一个全国排名视图，默认展示前 20 门店。

数据按一级板块和二级指标组织：

| 一级板块 | 二级指标 | 说明 |
| --- | --- | --- |
| 销售订单数量 | 销售订单数量 | 按销售归属门店统计订单数，不要求订单已核销。 |
| 核销订单数 | 本店卖出、本店核销数 | 销售归属门店是本店，实际核销门店也是本店的订单数。 |
| 核销订单数 | 本店卖出、他店核销数 | 销售归属门店是本店，实际核销门店是其他门店的订单数。 |
| 核销订单数 | 他店销售、在本店核销数 | 销售归属门店是其他门店，实际核销门店是本店的订单数。 |
| 订单收入 | 本店核销收入 | 在本店核销所产生的所有收入，包含本店销售本店核销收入，以及他店销售但在本店核销、扣除分佣后的收入。 |
| 订单收入 | 有效分佣收入 | 本店销售、他店核销所产生的分佣参考金额。 |

页面底部需要列出本页所有展示数据的计算口径，至少包括销售订单数量、三类核销订单数、本店核销收入、有效分佣收入、产品筛选口径和时间筛选口径。前端数字卡片或表格关键指标右上角应提供提示 icon，点击或悬停时展示对应数据口径。

### 5. 页面 2：单店月度分账看板

页面用于查看某一门店在指定月份、指定产品范围内的分账参考情况。页面顶部提供月份、门店、全部产品/特定服务产品筛选。

页面上方展示核心数字：

| 指标 | 时间口径 | 说明 |
| --- | --- | --- |
| 当期应收分佣 | 按到票或发票审核通过时间 | 本店销售、他店核销后，由销售店发票到票并审核通过确定，计入当期实际应收分佣。 |
| 可分佣总金额 | 按核销时间 | 本店销售、他店核销后，已经形成可分佣基础的分佣金额。 |
| 本店预计分出分佣参考额 | 按核销时间 | 他店销售、本店核销后，本店预计需要分出的分佣参考金额。 |

每个核心数字右上角应提供提示 icon，点击或悬停时展示该数字的数据口径。移动端可采用点击展开。

页面下方展示三块表格：

**应收分佣：本店卖出，他店核销**

| 商品类型 | 核销单数 | 订单实收金额 | 分佣比例 | 可分佣总金额 | 已到票单数 | 当期应收分佣 | 未到票待确认分佣 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |

**应付分佣：他店卖出，本店核销**

| 商品类型 | 单数 | 订单实收金额 | 分佣比例 | 应付分佣 |
| --- | ---: | ---: | ---: | ---: |

**不参与分佣：本店卖出，本店核销**

| 商品类型 | 单数 | 订单实收金额 |
| --- | ---: | ---: |

页面底部需要列出本页所有展示数据的计算口径，至少包括当期应收分佣、可分佣总金额、本店预计分出分佣参考额、应收分佣表各列、应付分佣表各列、不参与分佣表各列、产品筛选口径和月份筛选口径。月份筛选在不同指标中使用不同时间字段：可分佣总金额和应付分佣按核销时间归属月份，当期应收分佣按到票或发票审核通过时间归属月份。

### 6. 页面 3：门店月度数据明细表

页面用于呈现后端完整明细表，作为页面 1 和页面 2 汇总数据的证据来源。页面不直接展示“本店卖出、本店核销”“本店卖出、他店核销”“他店销售、在本店核销”等透视关系，而是展示销售归属门店、实际核销门店、是否核销、是否分佣等底层字段。

页面默认展示后端完整表，可通过筛选条件收窄数据范围。核心筛选和展示字段包括：

| 字段 | 说明 |
| --- | --- |
| 产品范围 | 全部产品或特定服务产品。 |
| 销售归属门店 | 订单销售归属的门店。 |
| 销售月份 | 订单销售发生月份，通常由下单时间或销售统计时间归属。 |
| 是否核销 | 标识订单或券是否已经核销；未核销数据不计入分佣状态计算。 |
| 实际核销门店 | 券实际完成核销的门店。 |
| 核销月份 | 核销发生月份，由核销时间归属。 |
| 是否分佣 | 已核销数据中，销售归属门店与实际核销门店一致则不分佣，不一致则分佣；未核销数据不参与该字段计算。 |
| 到票状态 | 未到票、已到票、审核通过、审核未通过。 |
| 退款状态 | 未退款、已退款、退款中。 |
| 订单 / 券搜索 | 支持按订单 ID 或券 ID 搜索。 |

从页面 2 跳转到页面 3 时，页面 3 不接收页面 2 的透视关系字段，而是接收并展示底层筛选条件：

| 页面 2 点击位置 | 页面 3 自动筛选 |
| --- | --- |
| 当期应收分佣 | 销售归属门店为当前门店、实际核销门店不是当前门店、到票状态为审核通过，月份按到票或审核通过时间归属。 |
| 可分佣总金额 | 销售归属门店为当前门店、实际核销门店不是当前门店、是否核销为是，月份按核销时间归属。 |
| 本店预计分出分佣参考额 | 实际核销门店为当前门店、销售归属门店不是当前门店、是否核销为是，月份按核销时间归属。 |
| 不参与分佣表 | 销售归属门店为当前门店、实际核销门店为当前门店、是否核销为是、是否分佣为否。 |

页面支持导出当前筛选后的明细数据，用作财务复核和线下对账依据。导出时应保留当前筛选条件，并在导出文件中包含销售归属门店、销售月份、是否核销、实际核销门店、核销月份、是否分佣、到票状态和退款状态等核心字段。

## 七、明确不做的系统动作

为了避免产品边界混淆，本产品第一阶段不承担以下动作：

- 不创建抖音交易订单。
- 不执行券核销。
- 不发起退款。
- 不开具或审核发票。
- 不执行付款、打款或资金划拨。
- 不生成正式财务凭证。
- 不替代集团财务的最终结算确认。

## 八、当前产品形态

当前阶段产品以本地 Python 脚本、CSV/JSON 中间文件和静态 HTML 分账看板为主：

- 数据采集、清洗和汇总由 Python 脚本完成。
- 路径、账号、SKU 等本地差异通过配置文件管理。
- 分账看板可生成为 HTML 文件，供协作者和业务人员查看。
- 分账参考结果仍依赖现有脚本计算和 CSV/JSON 中间文件。
- 此前独立推进的商品销售核销看板、每日发布流程和腾讯云 COS 上传流程已从当前主线迁出，不作为本产品当前主线能力。

后续演进方向是将数据存储迁移到云数据库，将看板能力服务化，由后端提供 API，前端负责展示、筛选和复核交互。飞书多维表格可作为人工校对、异常处理、门店映射维护和财务确认的协作界面，但不建议作为主数据库。

## 九、发票模块边界

发票上传、发票文件存储、发票信息识别和发票与订单/分账参考数据的关联规则尚未确定。第一阶段只预留发票相关状态或资料边界，不把发票模块作为主流程依赖。

页面如展示当期应收分佣，只读取财务已维护或导入的到票时间、发票审核状态和审核通过时间，不在系统侧完成发票上传、发票识别或发票审核流程。

后续如需建设发票模块，应单独确认：

- 发票文件存储位置。
- OCR 或结构化识别方案。
- 人工复核流程。
- 发票与订单、核销、退款、门店、分账参考记录的关联关系。
- 飞书是否承担发票人工校对界面。

## 十、产品价值

本产品的核心价值是把原本分散在抖音后台、CSV、Excel 和人工核对流程中的数据，整理为门店可查看、集团财务可复核的统一经营数据入口。它帮助门店理解自己的抖音订单履约和跨店核销情况，也帮助集团财务基于更完整的底层数据进行分账参考核对。
