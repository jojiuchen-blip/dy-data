# 后端开发规则

## 适用范围

适用于 `apps/api/`、`apps/worker/`、`src/dy_data/` 中的 FastAPI 接口、业务服务、数据访问、同步任务和权限逻辑。

## 开始前

1. 确认当前 Linear issue、验收标准和数据边界。
2. 读取相关 route、schema、dependency、service、model、migration 和测试，不从旧文档猜接口。
3. 涉及数据库时同时加载 `database-tasks.md`；涉及页面联调时同时加载 `frontend-tasks.md`。

## 实现规则

- 路由只处理 HTTP 输入输出、依赖注入和错误映射；可复用业务计算放入服务或领域层。
- Pydantic schema 明确请求、响应和可空性，禁止用无约束字典掩盖契约变化。
- 业务响应沿用当前 `{ data, definitions?, meta }` 包络；调整字段时同步后端测试、前端类型和 `docs/api-contract.md`。
- 业务接口默认通过 `get_current_user` 或更严格的管理员依赖鉴权。门店数据范围必须由后端校验，不能只靠前端隐藏。
- 金额、日期、订单号、门店 ID 和状态值沿用现有数据库与 schema 口径；金额计算不得引入浮点误差。
- 写操作明确事务边界、重复请求行为和失败回滚；同步、刷新、重建类任务必须可观察且尽量可重跑。
- 禁止在日志和异常中输出密钥、Cookie、数据库 URL、完整手机号或其他敏感数据。
- 不为通过单个页面临时硬编码门店、商品类型、账号或业务规则。

## 最小验证

- 为正常、空数据、无权限、非法输入和关键边界补充或更新 pytest。
- 接口变化至少验证真实应用路由、依赖和序列化结果，不只测孤立函数。
- 涉及数据库写入时验证提交后回读；涉及 worker 时验证失败状态和重试或恢复路径。
- 完成前运行相关测试，交付门禁运行 `python -m pytest`。

## 禁止行为

- 绕过认证依赖直接暴露业务或管理接口。
- 将数据库模型直接作为不受控的外部响应契约。
- 在没有迁移、回填和回滚判断时改变持久化字段语义。
- 只看到 HTTP 成功就宣称业务链路正确。
