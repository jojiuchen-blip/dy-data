# 架构整理说明

## 当前阶段

本仓库原本是个人本地脚本集合，数据文件、运行时和输出目录集中在作者本机。当前改造目标是先完成配置化，不改变核心业务口径，也不一次性移动所有脚本。

## 目标方向

后续架构建议分三层推进：

1. 采集层：订单、核销、退款、职人绑定信息仍由定时任务采集。
2. 数据层：云数据库作为主存储，飞书多维表格用于人工校对、异常处理和运营协作。
3. 服务层：后端提供看板 API，前端部署到服务器，不再依赖单个巨大静态 HTML 内嵌全部数据。

## 本次整理边界

- 新增 `src/dy_data` 公共模块。
- 新增 `config.example.json` 和 `requirements.txt`。
- 核心脚本改为读取统一配置。
- 暂不移动根目录脚本，避免破坏协作者现有命令和定时任务。

## 后续目录目标

```text
scripts/
  exports/
  dashboards/
  settlement/
  diagnostics/
  tasks/
src/
  dy_data/
docs/
tests/
```

迁移目录前，应先确认协作者环境可以用 `config.local.json` 跑通每日流程和主要导出流程。
