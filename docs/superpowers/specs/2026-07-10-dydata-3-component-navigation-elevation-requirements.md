# DYDATA-3 候选规范组件层级扩充需求

> 状态：Needs Decision。本文只定义候选规范扩充范围；得到用户确认前，不修改候选 HTML、候选 token 或任何业务运行时文件。

## 背景

V0.2 已完成品牌橙、黑白灰和基础组件预览，但指标卡、导航和阴影仍未形成完整组件契约：指标卡只有基础尺寸，导航没有独立定义，移动端底部一级导航缺少“数据表现”，阴影只有泛用值且部分仍继承旧绿色体系。

## 当前问题

- `MetricCard` 没有明确的监控、重点、语义和加载状态，容易把普通信息卡误用成指标卡。
- `NavigationItem` 没有跨桌面二级菜单、桌面一级模块导航和移动底部导航的统一尺寸、激活态、图标和文案规则。
- 移动端底部一级导航样板未包含“数据表现”入口，无法完整表达产品的四个核心模块。
- 阴影 token 只有 `shadow` 与 `shadowSoft`，并含旧绿色阴影色值；卡片、菜单、弹层、工作台和底部导航之间缺乏可解释的层级差异。

## 目标结果

- V0.2 候选规范完整定义 `MetricCard`、`NavigationItem` 和 `Elevation` 三个基础模块，并在 HTML 中以可见状态矩阵呈现。
- 组件颜色只使用候选橙黑灰 token 与既有成功、警告、错误、信息语义色；不向业务运行时写入任何候选值。
- 移动端候选样板展示四个可点击的一级入口：数据表现、结算、线索、后台。
- 阴影变为中性黑透明的有限层级，视觉上轻、准、可解释，白色表面始终搭配 1px 边线。

## 范围

### MetricCard

- 新增四种候选状态：`standard`、`primary`、`semantic`、`loading`。
- 固定内容顺序：指标名称、主数值、辅助说明；辅助说明不可重复主数值。
- `standard` 使用白色表面、极浅阴影和中性边线。
- `primary` 仅用于当前页面最重要的一个指标，使用 3px 深橙顶部强调线，不使用整卡橙色填充。
- `semantic` 仅用于成功、警告、错误、信息等业务状态，继续使用对应语义色顶部强调线。
- `loading` 保持最终卡片尺寸和内容区域，不使用跳动或装饰性动画。

### NavigationItem

- 新增独立导航组件规范，覆盖：桌面 Rail 一级模块、桌面顶部二级菜单、移动顶部二级菜单和移动底部一级导航。
- 同一模块在不同断点只改变布局，不改变图标语义或标签名称。
- 桌面一级模块为图标加文字，激活态用浅橙背景、深橙文字与品牌图标；桌面二级菜单用图标加文字和 2px 深橙底边。
- 移动端底部导航固定四项：`数据表现`、`结算`、`线索`、`后台`；每项最小点击高度 56px、图标 24px、标签 12px。激活项使用浅橙容器和深橙图标/文字，非激活项使用灰色。
- “数据表现”是经营总览/看板入口的候选命名和图标语义，本阶段仅出现在规范样板，不改路由或业务导航。

### Elevation

候选 token 使用中性黑 `rgb(24 24 24 / alpha)`，不再使用旧绿色阴影。定义五层：

| 层级 | 候选 token | 使用范围 |
| --- | --- | --- |
| 0 | `shadowNone` | 表格、普通内容区、Chip |
| 1 | `shadowCard` | 指标卡、筛选面板、普通信息卡 |
| 2 | `shadowPopover` | Select/Combobox 菜单、提示卡、轻量浮层 |
| 3 | `shadowDialog` | Dialog、ConfirmDialog、抽屉 |
| 4 | `shadowWorkbench` | 线索跟进工作台等高优先级覆盖层 |

候选值：

- `shadowNone`: `none`
- `shadowCard`: `0 2px 10px rgb(24 24 24 / 4%), 0 12px 28px rgb(24 24 24 / 4%)`
- `shadowPopover`: `0 10px 28px rgb(24 24 24 / 9%), 0 2px 8px rgb(24 24 24 / 5%)`
- `shadowDialog`: `0 18px 48px rgb(24 24 24 / 14%), 0 4px 14px rgb(24 24 24 / 7%)`
- `shadowWorkbench`: `0 30px 80px rgb(24 24 24 / 18%), 0 8px 24px rgb(24 24 24 / 9%)`

## 不做

- 不修改 `apps/web/src/design-tokens.css`、`apps/web/src`、路由或真实移动端导航。
- 不迁移已有业务页面的指标卡、导航或阴影。
- 不引入暗色模式、动效系统或新的图标库。
- 不改变成功、警告、错误、信息色的语义。

## 验收标准

- [ ] 候选 JSON 具有 `metricCard`、`navigation` 与五层 `shadow` 的机器可读规则。
- [ ] HTML 可见展示四种指标卡状态、四种导航上下文和五层阴影对照。
- [ ] 移动底部导航样板包含数据表现、结算、线索、后台四项，标签不截断，点击目标不低于 44px。
- [ ] 所有候选阴影使用中性黑透明值；不再出现旧绿色阴影色值。
- [ ] 桌面 1440、平板 768、手机 390 均无重叠、裁切或全局横向溢出。
- [ ] 设计规范测试通过，候选文件继续不被 `apps/web/src` 引用。

## 风险 / 依赖

- 该需求改变候选规范的组件语言，需要用户在 HTML 中确认视觉效果后，才可进入实现。
- 数据表现入口仅为样板信息架构，真实路由命名、权限和业务功能属于后续业务迁移范围。
- 本需求仍受 DYDATA-3 的阶段门禁约束，未得到“确认进入阶段 2”前不进入运行时。
