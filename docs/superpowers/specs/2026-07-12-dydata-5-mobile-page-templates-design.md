# DYDATA-5 V0.2 移动端页面骨架设计

## 目标

让 `candidate-v0.2.html` 在 390px 和 768px 下只展示真实可执行的移动端页面语义，不再把桌面 Shell、宽表和桌面分页压缩或裁切成伪移动页面。业务运行时文件不在本 issue 中修改。

## 断点策略

- `> 920px`：展示桌面 Shell、桌面明细工作台和移动端参考样板，便于完整对照。
- `641px - 920px`：隐藏桌面 Shell 和桌面宽表，展示两列移动端样板，并显示桌面样板仅在宽屏展示的说明。
- `<= 640px`：隐藏桌面 Shell 和桌面宽表，移动端样板改为单列。

桌面样板保持原始桌面结构，不缩放、不横向滚动、不参与窄屏布局。

## 移动端页面模板

### 线索明细

- 顶部二级导航：`线索看板 / 线索明细`。
- 页面标题、数据状态和总数。
- 折叠筛选按钮显示当前已选条件数量；已选条件以 FilterChip 摘要展示。
- 结果使用线索记录卡片，显示状态、轮次、联系方式、商品、最近跟进和生成时间。
- 主操作为全宽“查看详情”，触控高度不低于 44px。
- 分页固定在结果区之后，只保留上一页、可编辑页码和下一页。
- 底部一级导航固定为：数据表现、结算、线索、后台。

### 订单明细

- 顶部二级导航：`全国门店榜单 / 单店结算 / 订单明细`。
- 展示筛选器展开态：销售门店、月份、核销状态，以及清空和收起操作。
- 结果使用订单记录卡片，显示订单状态、核销状态、商品名称、完整门店名、订单编号、支付时间和金额。
- 分页规则与线索明细一致。
- 底部一级导航固定为四项，其中结算为当前模块。

## 组件与样式边界

- 新增样板 class 仅服务候选规范 HTML，不表示业务组件已经迁移完成。
- 复用既有颜色、字体、圆角、阴影和 44px 触控目标，不新增品牌值。
- 移动记录使用卡片映射，不渲染桌面 `<table>`。
- 筛选器、分页和底部导航不得出现逐字换行或局部横向裁切。

## 测试

- 静态测试校验桌面/移动样板分组、移动卡片、筛选状态、分页结构和四项底部导航。
- 先写失败测试，再实现 HTML/CSS。
- Playwright 在 390px 和 768px 检查：页面和移动样板横向溢出为 0，桌面样板不可见，移动卡片可见，控制台 0 errors。
- 1440px 检查桌面样板仍可见且原有页面骨架没有回归。
- 完整运行设计系统测试、全量 pytest、前端构建和 `git diff --check`。

## 非目标

- 不修改 `apps/web/src`。
- 不迁移实际线索页或订单页。
- 不改变 V0.2 色值、字体、组件契约或后端数据结构。
