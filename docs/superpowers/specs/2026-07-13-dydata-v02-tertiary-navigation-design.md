# dy-data V0.2 三级导航设计

## 目标

为 V0.2 视觉规范补充统一的三级导航模式，使二级页面合并、页面内稳定子页面和实体详情子页面拥有一致的信息层级。首个明确用例是将结算中心现有三个二级页面合并为一个二级页面，并在其中提供“全国门店榜单”“单店结算”“订单明细”三个三级入口。

本规格只定义视觉规范和组件契约，不迁移业务路由或页面。

## 信息层级

- 一级导航表示业务模块，例如数据表现、结算、线索、后台。
- 二级导航表示模块内的主要工作区或主要页面。
- 三级导航表示某个二级页面下稳定、可独立访问的子页面。
- 页面筛选条件、状态条件和临时展示模式不属于三级导航。

三级页面必须具有稳定 URL，支持刷新、深链接、浏览器前进后退和权限控制。仅改变同一数据展示方式的临时操作应使用 segmented control；缩小数据范围的操作应使用筛选器或 FilterChip。

## 选定方案

采用页面内水平下划线导航，不使用标题行分段控件，也不增加页面内左侧导航。

### 桌面端

- 放在页面标题和说明下方、筛选器或正文内容上方。
- 实体详情页放在实体标题和摘要下方、详情内容上方。
- 不进入全局顶部栏，不与二级菜单并排，不建立第三条全局导航栏。
- 普通页面随页面内容滚动，不独立 sticky。
- 数据工作台将表格滚动限制在结果区域，三级导航自然保留在表格滚动区域之外，不再叠加 sticky offset。

### 移动端

- 放在页面标题下方、内容上方。
- 使用单行横向滚动，不强制平均分配宽度，不截断名称。
- 当前项在进入页面时自动滚动到可视区域。
- 单项触控区域不低于 44px。
- 不与顶部二级导航同时 sticky，避免持续占用纵向空间。
- 不默认折叠为下拉菜单，使当前层级和相邻子页面保持可见。

## 视觉规则

- 容器不使用卡片、填充背景或阴影，只保留必要的底部分隔线。
- 桌面单项高度复用 38px 控件节奏；移动端最小高度为 44px。
- 默认项使用辅助文字色和中等字重。
- Hover 使用浅橙交互背景，不改变布局尺寸。
- 当前项使用深橙文字和 2px 品牌橙下划线，颜色不是唯一的当前态线索。
- Focus-visible 使用全局 focus ring，且不能被下划线覆盖。
- 建议每组 2–5 项。超过 5 项时重新梳理信息架构，不增加“更多”菜单掩盖层级问题。
- 名称使用用户可理解的业务词汇，并与页面标题、URL、权限文案和错误提示一致。

## 组件契约

建议组件命名为 `TertiaryNav`，使用导航语义而不是标签页语义。

```tsx
type TertiaryNavItem = {
  label: string;
  href: string;
  current?: boolean;
  disabled?: boolean;
};

type TertiaryNavProps = {
  label: string;
  items: TertiaryNavItem[];
};
```

渲染要求：

- 根元素使用 `<nav aria-label="...">`。
- 每个可用项目使用真实 `<a href="...">`。
- 当前项目使用 `aria-current="page"`。
- 不使用 `role="tablist"`、`role="tab"` 或无 URL 的点击 `span`。
- 禁用项目不应成为主要产品路径；若必须预告未开放功能，应有清楚文案且不可伪装为可访问页面。
- 键盘使用浏览器原生 Tab 顺序，不引入方向键标签页模式。

## 路由示例

结算中心合并后，三个三级页面应保持独立路径。最终路径以业务迁移计划为准，但必须满足同一二级工作区下的稳定层级，例如：

```text
/settlement/ranking
/settlement/stores
/settlement/orders
```

实体详情页采用同一组件，例如：

```text
/admin/accounts/:id/profile
/admin/accounts/:id/permissions
/admin/accounts/:id/activity
```

## V0.2 规范落地范围

- `candidate-v0.2.html` 的组件展厅增加 `TertiaryNav / 三级导航`。
- 同时展示结算中心页面型、账号详情型和移动端横向导航样张。
- 展示默认、Hover、Focus-visible、当前和禁用状态。
- `tokens.v0.2-candidate.json` 增加三级导航组件规则、尺寸、指示线和响应式约束。
- 页面模板补充可选三级导航层级，并明确它不参与表格 sticky offset。
- 决策记录写明三级菜单与筛选器、segmented control 的语义边界。
- 静态测试校验真实导航语义、`aria-current`、稳定 `href` 和移动端 44px 点击目标。

## 验收标准

- 人类可在 V0.2 HTML 中直接比较页面型、详情型和移动端三级导航。
- 三级导航在 1440px、768px 和 390px 宽度下无重叠或页面级横向溢出。
- 长名称不被省略，移动端仅导航容器自身可横向滚动。
- 当前态同时具有文字、字重和下划线线索，并满足浅色模式对比度要求。
- 规范示例使用语义链接和 `aria-current="page"`，不使用假按钮或假标签页。
- 表格冻结、顶部二级导航和三级导航之间不产生新的 sticky 交错。

## 非目标

- 本阶段不合并结算中心业务页面。
- 本阶段不修改现有路由、后端 API 或权限模型。
- 本阶段不实现第四级导航。
- 本阶段不增加暗色模式。
