# DYDATA-3 V0.2 规范页精修设计

## 目标

在不改变候选色值、组件契约和业务运行时的前提下，把 V0.2 从完整候选稿提升为可用于全站迁移决策的精修规范页。

## 范围

- 组件展厅改用自然高度和主次分区，短组件不再被同排长组件拉高。
- Dialog、Navigation、Elevation 等横向样板按内容宽度跨栏，减少半屏空位。
- 建立紧凑后台排版角色：正文 14px、说明 13px、元数据 12px；常用字重收敛为 500、600、700、800。
- 普通章节、token 卡和组件卡使用边线建立层级，不再默认投影；下拉、Dialog、工作台和固定导航保留对应 elevation。
- 增加桌面固定章节目录和移动端横向章节目录，链接到现有 section id。
- 统一展示文案与中英文组件命名，保留必要的代码标识，减少工程说明抢占视觉层级。

## 不做

- 不处理 DYDATA-5 中的移动端页面骨架和宽表样板适配。
- 不修改 `apps/web/src`、运行时 token、业务路由或后端接口。
- 不改变候选橙、语义色、控件高度、圆角和组件行为契约。
- 不引入字体、图标或构建依赖。

## 版面方案

采用单一主内容列加顶部固定章节导航。桌面端目录保持一行可扫描；窄屏时同一目录横向滚动。组件展厅使用 `align-items: start` 的自然高度网格，信息量较大的样板可跨两列。

## 排版方案

使用适合数据后台的紧凑比例，不追求营销页的大字号：

- H1：34px / 800
- H2：24px / 800
- H3：18px / 700
- 正文与控件：14px / 500-700
- 说明：13px / 500
- 元数据与代码标签：12px / 600

移动端不缩小正文和说明，只压缩 H1-H3。

## 层级方案

- Level 0：页面章节、组件卡、token 卡、表格和 Chip，使用 1px 边线。
- Level 1：确实需要分组的指标卡、移动记录卡和决策提示，使用极轻 shadowCard。
- Level 2：Select、Combobox、Popover。
- Level 3：Dialog、ConfirmDialog。
- Level 4：线索跟进工作台。

## 验收

- 1440px 下组件卡自然高度，无因同排内容差异造成的大块空白。
- 规范页具备可键盘访问的章节目录，当前章节链接有明确反馈。
- 正文和组件说明不低于 13px；12px 只用于元数据或代码标签。
- 普通 token 卡和组件卡不使用 box-shadow。
- 390 / 768 / 1440 下无全局横向溢出、唯一 H1、控制台 0 errors。
- 现有设计系统文档和 enforcement 测试通过，业务运行时文件保持不变。
