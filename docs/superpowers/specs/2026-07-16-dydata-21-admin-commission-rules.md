# 【管理后台】分佣规则修改

日期：2026-07-16

状态：需求规则已确认，待协作者读取 Linear 后进入生产功能开发

核心关联：DYDATA-21

分支：`codex/dydata-21-admin-commission-rules`

## 1. 目标

在管理后台重构现有 SKU 单一分佣比例配置，建立可按月份追溯的推广费与管理服务费双比例规则。

本规格只描述管理后台规则、接口、数据和结算计算需要满足的业务契约，不属于订单分佣前端页面视觉改造。

## 2. 术语

前端、接口、导出和文档统一使用：

- 推广费比例
- 应收推广费
- 管理服务费比例
- 应扣管理服务费

不再使用含义不明确的“单一分佣比例”作为新业务规则名称。

## 3. 规则模型

规则唯一维度为：

```text
产品范围＋商品类型＋生效月份
```

规则结构分为两层：

1. SKU 映射层：`SKU → 产品范围 → 商品类型`
2. 商品费用规则层：`产品范围＋商品类型＋生效月份 → 推广费比例＋管理服务费比例`

同一产品范围、同一商品类型、同一生效月份只能存在一个有效规则版本。同类 SKU 继承同一组比例，不允许在同一规则版本内逐 SKU 设置不同费率。

## 4. 双比例行为

- 推广费比例和管理服务费比例分别保存。
- 新建规则时可以使用一项数值初始化另一项，但初始化完成后两项独立编辑，修改一项不能覆盖另一项。
- 比例范围为 `0%～100%`，最多两位小数。
- 明确配置 `0%` 表示规则已配置，费用结果为零。
- 空值表示待配置，不等同于 `0%`，不能被接口或导入流程静默转换。
- 管理服务费比例低于推广费比例时显示醒目提醒，但不阻止保存。
- 修改规则必须创建新的生效月份版本，不能覆盖已被历史账期引用的版本。
- 已锁账月份引用的规则和计算快照不可直接修改。

## 5. 计算方向

- 应收推广费归属于销售门店，按有效核销券金额乘推广费比例逐券计算。
- 应扣管理服务费归属于实际核销门店，按有效核销券金额乘管理服务费比例逐券计算。
- 每张券分别计算并四舍五入到分，再进行汇总。
- 同店销售并核销时，同一张有效券可以同时产生应收推广费和应扣管理服务费。
- 退款、关闭、撤销券不计算费用。
- 已核销但券级金额缺失时标记金额异常，不回退使用订单整单金额。

## 6. 销售总金额口径

销售总金额只统计推广费比例已配置的商品：

- 推广费比例为 `0%` 时，视为已配置，销售金额继续计入。
- 推广费比例为空时，视为待配置，对应销售金额暂不计入。
- 管理服务费比例是否配置，不影响销售总金额是否计入。

页面必须展示说明：

> 销售总金额仅统计已配置推广费比例的商品；比例为 0% 视为已配置，待配置商品暂不纳入。

## 7. 现有规则迁移边界

- 当前 `commission_rate` 是旧单比例字段，不能直接作为新业务长期模型。
- 本轮不预设精诚养车的推广费比例和管理服务费比例，也不默认写成 `10%＋10%`。
- 精诚养车及其他现有产品的初始双比例，必须在整体规则、切换月份和历史处理方案确认后单独决定。
- 历史已锁账结果不得因字段迁移被自动重算。
- 未锁账月份是否按新规则重建，必须通过差异预览和明确切换决策后执行。

## 8. 管理后台能力

管理后台至少需要支持：

- 按产品范围、商品类型、生效月份查看规则。
- 分别查看和编辑推广费比例、管理服务费比例。
- 区分已配置、`0%` 和待配置状态。
- 查看当前有效版本及历史版本。
- 保存前展示实际字段差异。
- 保存后记录规则版本、操作时间和变更前后值。
- 规则变更后仅对允许重建的未锁账月份触发结算重建。

## 9. 批量比例导入边界

批量导入是独立于 DYDATA-21 核心单条规则能力的后续需求，需要单独 Linear issue。

已确认的边界：

- 导入对象是推广费比例和管理服务费比例，不是订单、券或门店月度金额调整。
- 原始订单和系统计算结果不可通过导入直接覆盖。
- 导入应先校验并展示规则导入前后差异，再确认生效。
- 需要保存原文件、文件哈希、批次信息和每行处理结果。
- 已生效批次不能删除；错误需通过后续更正规则版本处理。
- 角色、账号和权限配置不在该需求中定义。

待独立需求确认：

- 模板以“产品范围＋商品类型＋生效月份”为主键，还是保留 `SKU_ID` 作为校验列。
- 批次是否必须全量原子生效。
- 生效前取消、已生效更正和结算重建的具体状态机。

## 10. 不做

- 不修改订单分佣前端页面视觉。
- 不在本规格分支实现生产 API、数据库迁移或 React 页面。
- 不配置系统角色、财务角色、账号或权限矩阵。
- 不定义订单或券级人工金额调整。
- 不直接修改历史锁账数据。
- 不在未确认整体规则前写入精诚养车初始双比例。
- 不在本分支合并 `main` 或部署生产。

## 11. 开发验收标准

- 双比例以“产品范围＋商品类型＋生效月份”唯一定位。
- 两项比例可以独立保存，修改一项不覆盖另一项。
- `0%`、待配置和非零比例在数据库、API、页面与计算中严格区分。
- 历史版本可查询，已锁账月份引用版本不可改写。
- 销售总金额遵守本规格第 6 节口径并展示说明。
- 推广费与管理服务费分别按销售门店和核销门店计算。
- 同店有效核销券可以同时产生两项费用。
- 迁移与重建必须提供规则数量、待配置数量、样本金额和差异记录。
- 相关 API、Worker、数据库和前端测试通过。
